常用 Docker 命令技术文档 ​

简介 ​

Docker 是一个开源的应用容器引擎，基于 Go 语言并遵从 Apache2.0 协议开源。Docker 可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。

容器是完全使用沙箱机制，相互之间不会有任何接口。Docker 的核心组件包括：

  • 镜像（Image）：Docker 的只读模板，用于创建容器
  • 容器（Container）：镜像的运行实例，可以被启动、停止、删除等
  • 仓库（Repository）：集中存储镜像的地方，如 Docker Hub
  • 网络（Network）：用于容器间通信的网络配置
  • 卷（Volume）：用于持久化存储数据的机制

Docker 广泛应用于以下场景：

  • 微服务架构：将应用拆分为多个独立的服务
  • 持续集成/持续部署：自动化构建、测试和部署
  • 开发环境一致性：确保开发、测试和生产环境一致
  • 云原生应用：与 Kubernetes 等编排工具集成
  • 快速部署：简化应用的部署和管理

核心概念 ​

镜像（Image） ​

镜像是一个只读的模板，包含了运行应用所需的所有文件和依赖。镜像可以从 Docker Hub 拉取，也可以通过 Dockerfile 构建。

容器（Container） ​

容器是镜像的运行实例，包含了应用及其运行环境。容器可以被启动、停止、删除等操作。

仓库（Repository） ​

仓库是集中存储镜像的地方，分为公共仓库（如 Docker Hub）和私有仓库。

网络（Network） ​

Docker 网络用于容器间通信，支持多种网络模式：

  • bridge：默认网络模式，容器通过桥接网络通信
  • host：容器直接使用宿主机网络
  • overlay：用于跨主机通信
  • macvlan：为容器分配 MAC 地址
  • none：禁用网络

卷（Volume） ​

卷用于持久化存储数据，即使容器被删除，数据也不会丢失。支持命名卷和绑定挂载。

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 更新包索引
sudo apt update

# 安装必要的包
sudo apt install apt-transport-https ca-certificates curl gnupg lsb-release

# 添加 Docker 官方 GPG 密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 添加 Docker 仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 更新包索引
sudo apt update

# 安装 Docker Engine
sudo apt install docker-ce docker-ce-cli containerd.io

# 启动 Docker 服务
sudo systemctl start docker

# 设置开机自启
sudo systemctl enable docker

# 将当前用户添加到 docker 组（避免使用 sudo）
sudo usermod -aG docker $USER
# 注销并重新登录以使更改生效

Docker 容器操作 ​

运行容器 ​

bash
# 基本运行
docker run nginx

# 后台运行并指定名称
docker run -d --name web-server nginx

# 交互式运行
docker run -it --name ubuntu-container ubuntu:20.04 /bin/bash

# 端口映射
docker run -d -p 8080:80 --name web nginx

# 多个端口映射
docker run -d -p 8080:80 -p 443:443 --name web-with-ssl nginx

# 绑定到特定IP
docker run -d -p 127.0.0.1:8080:80 --name web-local nginx

# 随机端口映射
docker run -d -P --name web-random nginx

# 挂载卷
## 绑定挂载

# 设置环境变量
docker run -d -e NGINX_HOST=localhost -e NGINX_PORT=80 --name web-env nginx

# 设置重启策略
docker run -d --restart unless-stopped --name web-restart nginx

# 资源限制
docker run -d --memory 512m --cpus 0.5 --name web-limited nginx

Docker Compose ​

Docker Compose 是一个用于定义和运行多容器 Docker 应用程序的工具。通过 Compose，可以使用 YAML 文件来配置应用程序的服务，然后使用一个命令即可从配置中创建并启动所有服务。

Docker Compose 的主要特点包括：

  • 简化多容器应用的部署
  • 通过单一配置文件管理所有服务
  • 支持服务间依赖关系定义
  • 提供环境变量管理
  • 支持卷和网络配置
  • 便于开发、测试和生产环境的一致性

# 添加执行权限
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装

语法 ​

Docker Compose 命令的基本语法结构：

docker-compose [options] [command] [arguments]
1

常用选项：

  • -f, --file FILE: 指定配置文件路径
  • -p, --project-name NAME: 指定项目名称
  • --env-file FILE: 指定环境变量文件
  • -v, --version: 显示版本信息
  • -h, --help: 显示帮助信息

容器执行和交互 ​

bash
# 在运行的服务中执行命令
docker-compose exec service_name command

# 在服务中启动交互式 shell
docker-compose exec -it service_name /bin/bash

# 在服务中运行命令（不进入容器）
docker-compose run service_name command

# 在服务中运行命令并删除容器
docker-compose run --rm service_name command

# 在服务中运行命令并覆盖入口点
docker-compose run --entrypoint command service_name

# 在服务中运行命令并设置环境变量
docker-compose run -e ENV_VAR=value service_name command

services:

      MYSQL_PASSWORD: wordpress

高级配置选项 ​

yaml
version: '3.8'

services:
  web:
    # 构建镜像
    build:
      context: .
      dockerfile: Dockerfile.web
      args:
        - BUILD_ARG=value
    
    # 端口映射
    ports:
      - "80:80"
      - "443:443"
    
    # 卷映射
    volumes:
      - ./app:/app
      - static_volume:/static
      - /host/path:/container/path:ro  # 只读
    
    # 环境变量
    environment:
      - ENV_VAR=value
    env_file:
      - .env
    
    # 重启策略
    restart: unless-stopped
    
    # 依赖关系
    depends_on:
      - redis
      - db
    
    # 健康检查
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
    
    # 资源限制
    deploy:
      resources:
        limits:
          cpus: '0.50'
          memory: 512M
        reservations:
          cpus: '0.25'
          memory: 256M
    
    # 网络
    networks:
      - frontend
      - backend

  redis:
    image: redis:alpine
    networks:
      - backend
    command: redis-server --appendonly yes

  db:
    image: postgres:13
    environment:
      POSTGRES_DB: mydb
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
  static_volume:

networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge

实用技巧 ​

资源限制 ​

bash
# 资源限制
docker run -m 512m --memory-swap 1g nginx    # 限制内存使用
docker run -c 512 nginx                      # 限制CPU使用

# 查看资源使用情况
docker stats --no-stream

# 环境变量
docker run -e ENV_VAR=value nginx
docker run --env-file .env nginx

# 安全操作
docker run --read-only nginx                 # 只读根文件系统
docker run --user 1000:1000 nginx            # 指定用户运行

最佳实践 ​

  1. 使用 .dockerignore 文件：避免将不必要的文件添加到镜像中
  2. 最小化层数：合并多个命令到一个 RUN 指令中
  3. 使用具体标签：避免使用 latest 标签
  4. 多阶段构建：减少最终镜像大小
  5. 安全扫描：定期扫描镜像漏洞
  6. 健康检查：在 Dockerfile 中添加 HEALTHCHECK 指令
  7. 资源限制：为容器设置适当的资源限制
