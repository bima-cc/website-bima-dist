CAS Overlay 开发版文档 ​

项目简介 ​

CAS (Central Authentication Service) 是 Apereo 基金会维护的企业级开源单点登录系统，为企业应用提供统一的身份认证与授权机制。本项目基于不同版本的 CAS Overlay 进行定制化开发，提供完整的开发基座、配置模板和部署方案，旨在帮助开发者快速构建、定制和部署符合企业需求的 CAS 认证服务。

版本选择 ​

根据您的技术栈和需求，选择适合的 CAS Overlay 版本：

CAS Overlay 7.3.x 系列 ​

  • JDK 版本：21
  • 主要特性：最新版本，支持现代 Java 特性，架构更加先进
  • 构建工具：
    • Maven 版本
    • Gradle 版本

CAS Overlay 6.6.x 系列 ​

  • JDK 版本：17
  • 主要特性：稳定版本，广泛应用于生产环境
  • 构建工具：
    • Maven 版本
    • Gradle 版本

CAS Overlay 5.3.x 系列 ​

  • JDK 版本：8+
  • 主要特性：经典版本，适用于旧环境
  • 构建工具：
    • Maven 版本
    • Gradle 版本

核心功能 ​

所有版本的 CAS Overlay 开发版都包含以下核心功能：

  • 架构转型：从传统 WAR 包架构升级为现代 JAR 包架构，简化部署流程，更好适配微服务环境和容器化部署
  • 数据库集成：集成 MyBatis 数据库访问框架，实现与各类关系型数据库的无缝对接
  • 自定义类启动CAS服务：提供 CasWebApplication 主启动类，支持开发者进行本地Debug启动和日常开发中的断点调试
  • 自定义认证：实现灵活的身份验证逻辑，支持自定义用户名密码的登录校验规则，同时支持oauth2.0认证
  • 自定义返回值：实现自定义的认证返回值格式，支持自定义返回用户信息、角色信息等任意数据
  • 分布式集群部署：基于 Redis 支持分布式多节点部署，实现高可用性和可扩展性
  • 安全加固：关闭 CAS 默认账号密码登录，强制使用自定义认证类，提升系统安全性
  • 资源提取：全量提取前端静态资源和模板，方便开发者根据企业品牌定制前端界面
  • 多语言支持：内置中文、英文等多语言支持，满足国际化需求（示例）
  • 容器化部署：内置DockerFile和Jenkins部署脚本，方便开发者进行容器化部署和持续集成

快速开始 ​

  1. 选择版本：根据您的 JDK 版本和构建工具偏好选择合适的 CAS Overlay 版本
  2. 查看文档：点击上方对应版本的链接查看详细文档
  3. 环境准备：按照文档要求准备 JDK、数据库等环境
  4. 数据库初始化：执行提供的数据库初始化脚本
  5. 配置修改：根据您的环境修改数据库连接等配置
  6. 构建运行：使用 Maven 或 Gradle 构建并运行项目
  7. 访问服务：通过 https://服务器IP:8443/cas/login 访问 CAS 服务

注意事项 ​

  • 安全提示：首次登录后请修改默认管理员密码
  • 证书警告：测试环境下访问会提示证书警告，可忽略或添加信任
  • 网络要求：构建过程需要访问 Maven 中央仓库，确保网络连接正常
  • 生产环境：在生产环境中使用前，请进行充分的测试和验证
