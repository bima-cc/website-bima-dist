常用 Node.js 命令技术文档 ​

简介 ​

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境，允许开发者使用 JavaScript 进行服务器端编程。Node.js 采用事件驱动、非阻塞 I/O 模型，使其轻量且高效，非常适合数据密集型实时应用程序。

Node.js 的核心特性包括：

  • 单线程事件循环模型
  • 非阻塞 I/O 操作
  • 丰富的内置模块
  • 强大的包管理器 npm
  • 跨平台兼容性

Node.js 广泛应用于 Web 服务器、API 服务、实时应用、微服务架构和工具开发等领域。

安装方法 ​

使用官方安装包 ​

Windows/macOS ​

  1. 访问 https://nodejs.org/
  2. 下载 LTS (长期支持) 版本
  3. 运行安装程序并按照向导完成安装

使用包管理器 ​

Linux (Ubuntu/Debian) ​

bash
# 使用 NodeSource 仓库（推荐）
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

# 或使用 snap
sudo snap install node --classic

# 验证安装
node --version
npm --version

使用 Node 版本管理器 ​

nvm (Node Version Manager) - Linux/macOS ​

bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 重启终端或运行
export NVM_DIR="$([ -z "${XDG_CONFIG_HOME-}" ] && printf %s "${HOME}/.nvm" || printf %s "${XDG_CONFIG_HOME}/nvm" )"

# 安装并使用 Node.js LTS 版本

npm 包管理命令 ​

bash
# 初始化新的 npm 项目
npm init
npm init -y                        # 使用默认设置

# 安装包
npm install package_name           # 安装包
npm install package_name@version   # 安装特定版本
npm install package_name --save-dev  # 安装为开发依赖
npm install package_name --save    # 安装为生产依赖（默认）
npm install package_name --global  # 全局安装
npm install                         # 安装 package.json 中的所有依赖

# 简写形式
npm i package_name                 # 安装
npm i -g package_name              # 全局安装
npm i -D package_name              # 开发依赖
npm i -S package_name              # 生产依赖

# 卸载包
npm uninstall package_name
npm uninstall package_name --save
npm uninstall package_name --save-dev
npm uninstall -g package_name      # 全局卸载

# 简写形式
npm un package_name
npm rm package_name
npm r package_name

# 更新包
npm update package_name            # 更新特定包
npm update                         # 更新所有包
npm update -g package_name         # 更新全局包

# 查看包信息
npm list                          # 查看本地包
npm list -g                       # 查看全局包
npm list package_name             # 查看特定包
npm list --depth=0               # 仅显示顶层包

# 搜索包
npm search keyword
npm search package_name

# 查看包详情
npm info package_name
npm view package_name
npm show package_name

# 查看过期包
npm outdated
npm outdated package_name

# 清理缓存
npm cache clean --force
npm cache verify

# 查看 npm 配置
npm config list
npm config get key
npm config set key value

# 运行脚本
npm run script_name               # 运行 package.json 中定义的脚本
npm start                         # 运行 start 脚本
npm test                          # 运行 test 脚本
npm run build                     # 运行 build 脚本

版本管理技巧 ​

bash

nvm install node              # 安装最新版本
nvm install --lts             # 安装最新 LTS 版本
nvm use node                  # 切换到最新版本

nvm alias default node        # 设置默认版本
nvm list                      # 列出所有已安装版本

# 在项目根目录创建 .nvmrc 文件
echo "lts/*" > .nvmrc         # 指定使用 LTS 版本

最佳实践 ​

  1. 使用 LTS 版本：始终使用 Node.js 的长期支持版本
  2. 版本锁定：在 package.json 中锁定依赖版本
  3. 安全审计：定期运行 npm audit 检查安全漏洞
  4. 语义化版本控制：遵循 SemVer 规范管理版本
  5. 使用 .nvmrc 文件：在项目中指定 Node.js 版本
  6. 区分依赖类型：正确使用 dependencies 和 devDependencies
  7. 使用锁文件：保留 package-lock.json 确保重现性
  8. 环境变量：使用环境变量管理配置
  9. 错误处理：妥善处理异步操作中的错误
  10. 性能监控：监控应用性能和内存使用
  11. 版本管理：使用 nvm 或类似的工具管理 Node.js 版本
  12. 依赖更新：定期更新依赖包并测试兼容性
