常用 PostgreSQL 命令技术文档 ​

简介 ​

PostgreSQL（简称 Postgres）是一个功能强大的开源对象关系型数据库系统，以其可靠性、功能强大和性能优越而闻名。PostgreSQL 支持复杂查询、外键、触发器、视图、事务完整性、多版本并发控制（MVCC）等高级功能，被广泛应用于各种规模的企业级应用中。

PostgreSQL 的主要特点包括：

  • 高度 SQL 标准兼容性
  • 支持复杂查询和数据类型
  • 丰富的扩展生态系统
  • 强大的并发控制机制
  • 支持 JSON、XML、数组等非结构化数据类型
  • 完全支持 ACID 特性
  • 支持存储过程和函数

PostgreSQL 广泛应用于 Web 应用、数据仓库、地理信息系统、科学计算和企业级应用等领域。

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 更新包索引
sudo apt update

# 安装 PostgreSQL 和附加组件
sudo apt install postgresql postgresql-contrib

# 启动 PostgreSQL 服务
sudo systemctl start postgresql

# 设置开机自启
sudo systemctl enable postgresql

# 检查 PostgreSQL 服务状态
sudo systemctl status postgresql

# 验证安装
sudo -u postgres psql -c "SELECT version();"

语法 ​

PostgreSQL 命令的基本语法结构：

psql 客户端语法 ​

psql [option...] [dbname [username]]
1

SQL 语句语法 ​

COMMAND [CLAUSES] [CONDITIONS];
1

常用 psql 选项：

  • -h, --host=HOSTNAME: 指定主机名
  • -p, --port=PORT: 指定端口号
  • -U, --username=USERNAME: 指定用户名
  • -d, --dbname=DBNAME: 指定数据库名
  • -c, --command=COMMAND: 执行 SQL 命令
  • -f, --file=FILENAME: 从文件执行 SQL 命令
  • -W, --password: 强制密码提示

表操作 ​

sql

\d table_name

-- 创建表
CREATE TABLE users (
    id SERIAL PRIMARY KEY,

    email VARCHAR(255) UNIQUE,

ALTER TABLE table_name RENAME COLUMN old_name TO new_name;

-- 重命名表
ALTER TABLE old_name RENAME TO new_name;

-- 查看表相关信息

数据操作 ​

sql
-- 插入数据
INSERT INTO users (name, email, age) VALUES ('John Doe', 'john@example.com', 30);

DELETE FROM users WHERE id = 1;
DELETE FROM users WHERE age < 18;

-- 条件查询
SELECT * FROM users WHERE name LIKE '%John%';
SELECT * FROM users WHERE age BETWEEN 18 AND 65;
SELECT * FROM users WHERE email IN ('john@example.com', 'jane@example.com');

备份和恢复 ​

bash
# 使用 pg_dump 备份单个数据库
pg_dump -U username -h hostname database_name > backup.sql

# 备份所有数据库
pg_dumpall -U postgres > all_databases.sql

# 备份特定表
pg_dump -U username database_name -t table_name > table_backup.sql

# 备份结构（不包含数据）
pg_dump -U username database_name --schema-only > structure.sql

# 备份数据（不包含结构）
pg_dump -U username database_name --data-only > data_only.sql

# 使用 pg_restore 恢复（针对自定义格式备份）
pg_restore -U username -d database_name backup.dump

# 从 SQL 文件恢复
psql -U username -d database_name < backup.sql

常用配置优化 ​

bash
# 编辑 PostgreSQL 配置文件
sudo nano /etc/postgresql/*/main/postgresql.conf  # Ubuntu/Debian
sudo nano /var/lib/pgsql/data/postgresql.conf    # CentOS/RHEL

# 常见性能优化配置
# 内存相关
shared_buffers = 256MB          # 推荐设置为系统内存的 25%
effective_cache_size = 1GB      # 推荐设置为系统内存的 50-75%
work_mem = 4MB                  # 每个内部排序操作和哈希表使用的内存量
maintenance_work_mem = 64MB     # 维护操作（如 VACUUM、CREATE INDEX）可用的最大内存量

# WAL 相关
wal_buffers = 16MB              # WAL 缓冲区大小
checkpoint_segments = 32        # 检查点之间的 WAL 文件数量（PostgreSQL < 9.5）
max_wal_size = 1GB              # 最大 WAL 大小（PostgreSQL >= 9.5）
checkpoint_completion_target = 0.9  # 检查点完成目标

# 并发相关
max_connections = 100           # 最大连接数

最佳实践 ​

  1. 合理设计数据库模式：根据业务需求设计合适的数据类型和约束
  2. 建立适当的索引：为经常查询的字段建立索引，但避免过度索引
  3. 使用连接池：应用层面使用连接池管理数据库连接
  4. 定期备份：制定合理的备份策略并定期验证备份有效性
  5. 安全配置：配置适当的用户权限，限制网络访问，使用 SSL 加密
  6. 性能监控：监控慢查询、连接数、内存使用等关键指标
  7. 参数调优：根据服务器硬件和应用负载调整 PostgreSQL 配置参数
  8. 定期维护：执行 ANALYZE、VACUUM 等维护操作
  9. 使用事务：在需要数据一致性的操作中使用事务
  10. 版本更新：定期更新 PostgreSQL 版本以获得新功能和安全补丁
  11. 分区表：对于大表考虑使用分区表以提高查询性能
  12. 读写分离：在高并发场景下考虑使用主从复制和读写分离
