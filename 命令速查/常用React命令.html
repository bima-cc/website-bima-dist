简介 ​

React 是一个用于构建用户界面的 JavaScript 库，由 Facebook 开发和维护。React 采用组件化的思想，允许开发者将复杂的 UI 拆分成独立、可复用的组件。React 的核心理念是"一次学习，随处编写"，它专注于构建应用的视图层，可以与其它库或框架很好地结合使用。

React 的主要特点包括：

  • 组件化架构：将 UI 拆分成独立的、可复用的组件
  • 虚拟 DOM：高效的 DOM 操作和更新机制
  • 单向数据流：清晰的数据流向和状态管理
  • JSX 语法：JavaScript 与 HTML 结合的语法糖
  • 强大的生态系统：丰富的第三方库和工具
  • 高性能：批量更新和差异算法优化渲染
  • 灵活性：可与现有应用集成，也可构建全新应用

React 广泛应用于现代 Web 应用开发，从简单的页面组件到复杂的单页应用（SPA）和服务器端渲染（SSR）应用，都是 React 的典型应用场景。配合 React Router、Redux/MobX 等工具，可以构建功能完善的现代化前端应用。

语法 ​

Create React App 命令的基本语法结构：

npx create-react-app [project-name] [options]
1

Vite 项目命令的基本语法结构：

npm run [script-name] [options]
1

常用选项：

  • --template <template>: 指定项目模板（react, react-ts, vue, vue-ts 等）
  • --use-npm, --use-yarn, --use-pnpm: 指定包管理器
  • --page, --spa: 指定项目类型
  • --eslint, --prettier: 配置代码质量工具
  • --git: 初始化 Git 仓库

React 函数组件示例 ​

jsx
import React, { useState, useEffect } from 'react';
import PropTypes from 'prop-types';

// 函数组件示例
const MyComponent = ({ title, initialCount = 0 }) => {
  const [count, setCount] = useState(initialCount);
  const [data, setData] = useState(null);

  // 模拟数据获取
  useEffect(() => {
    const fetchData = async () => {
      try {
        // 模拟 API 调用
        setTimeout(() => {
          setData({ message: 'Hello from API!' });
        }, 1000);
      } catch (error) {
        console.error('Error fetching data:', error);
      }
    };

    fetchData();
  }, []);

  const increment = () => {
    setCount(count + 1);
  };

  const decrement = () => {
    setCount(count - 1);
  };

  return (
    <div className="my-component">
      <h1>{title}</h1>
      <p>Count: {count}</p>
      <div>
        <button onClick={increment}>+</button>
        <button onClick={decrement}>-</button>
      </div>
      {data && <p>Data: {data.message}</p>}
    </div>
  );
};

// PropTypes 定义
MyComponent.propTypes = {
  title: PropTypes.string.isRequired,
  initialCount: PropTypes.number
};

export default MyComponent;

实用脚本示例 ​

bash
#!/bin/bash
# React 项目构建脚本

# 检查是否为 React 项目
if [ ! -f "package.json" ]; then
    echo "当前目录不是 React 项目"
    exit 1
fi

# 检查是否包含 React 依赖
if ! grep -q '"react"' package.json; then
    echo "当前项目不是 React 项目"
    exit 1
fi

# 检查依赖是否安装
if [ ! -d "node_modules" ]; then
    echo "安装依赖..."
    npm install
fi

# 运行代码检查
echo "运行 ESLint 检查..."
npm run lint

if [ $? -ne 0 ]; then
    echo "ESLint 检查失败，请修复问题后重试"
    exit 1
fi

# 运行格式检查
echo "运行 Prettier 检查..."
npm run format

# 运行测试
echo "运行测试..."
npm test -- --watchAll=false

if [ $? -ne 0 ]; then
    echo "测试失败，请修复问题后重试"
    exit 1
fi

# 构建项目
echo "构建项目..."
npm run build

if [ $? -eq 0 ]; then
    echo "构建成功！"
    echo "构建产物位于 build/ 目录"
else
    echo "构建失败！"
    exit 1
fi

最佳实践 ​

  1. 使用函数组件和 Hooks：优先使用函数组件和 React Hooks 而不是类组件
  2. 组件设计：遵循单一职责原则，保持组件简洁和可复用
  3. 状态管理：小型项目使用 useState/useReducer，大型项目使用 Context API 或 Redux Toolkit
  4. 路由设计：使用 React Router 实现客户端路由
  5. 代码分割：使用 React.lazy 和 Suspense 实现代码分割
  6. TypeScript：在大型项目中使用 TypeScript 提升代码质量
  7. 测试驱动：编写单元测试和集成测试
  8. 性能监控：使用 React DevTools Profiler 监控性能
  9. 安全性：防范 XSS 攻击，验证用户输入
  10. 文档化：为组件和功能编写清晰的文档
