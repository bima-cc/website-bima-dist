常用 Gradle 命令技术文档 ​

简介 ​

Gradle 是一个基于 Apache Ant 和 Maven 概念的项目自动化构建工具。它使用一种基于 Groovy 或 Kotlin 的特定领域语言（DSL）来声明项目设置，而不是传统的 XML 形式。Gradle 支持多语言开发，包括 Java、Scala、Android、C/C++ 等，并提供灵活的构建配置和强大的依赖管理功能。

Gradle 的主要特点包括：

  • 灵活的构建脚本：使用 Groovy 或 Kotlin DSL 编写构建脚本
  • 强大的依赖管理：支持多种仓库类型和依赖解析策略
  • 多项目构建：支持复杂的多模块项目结构
  • 增量构建：只构建发生变化的部分，提高构建效率
  • 构建缓存：缓存构建结果以加速重复构建
  • 并行执行：支持并行执行任务以提高构建速度
  • 可扩展性：通过插件系统扩展功能

Gradle 广泛应用于 Android 应用开发、Java 企业级应用、微服务架构以及现代化软件开发项目中，是目前最流行的构建工具之一。

手动安装 ​

  1. 访问 https://gradle.org/releases/
  2. 下载二进制发行版
  3. 解压到指定目录（如 C:\gradle）
  4. 将 C:\gradle\bin 添加到系统 PATH 环境变量
  5. 验证安装：gradle --version

使用 Gradle Wrapper（推荐方式） ​

bash

gradle wrapper

# 使用 Wrapper 运行 Gradle 命令（跨平台兼容）
./gradlew build          # Linux/macOS
.\gradlew.bat build      # Windows

# 指定 Gradle 版本创建 Wrapper
gradle wrapper --gradle-version 8.5

语法 ​

Gradle 命令的基本语法结构：

gradle [task...] [option...]
1

常用选项：

  • -g, --gradle-user-home <dir>: 指定 Gradle 用户主目录
  • -b, --build-file <file>: 指定构建文件
  • -c, --settings-file <file>: 指定设置文件
  • -D, --system-prop <name>=<value>: 设置系统属性
  • -I, --init-script <file>: 指定初始化脚本
  • -P, --project-prop <name>=<value>: 设置项目属性
  • -S, --full-stacktrace: 显示完整的堆栈跟踪
  • -i, --info: 设置日志级别为 info
  • -d, --debug: 设置日志级别为 debug
  • -q, --quiet: 设置日志级别为 quiet
  • --console <type>: 指定控制台输出类型（plain, auto, rich）
  • --parallel: 并行执行构建
  • --refresh-dependencies: 刷新依赖项
  • --offline: 离线模式运行

实用脚本示例 ​

bash
#!/bin/bash
# Gradle 构建脚本示例

# 检查是否为 Gradle 项目
if [ ! -f "build.gradle" ] && [ ! -f "build.gradle.kts" ]; then
    echo "当前目录不是 Gradle 项目"
    exit 1
fi

# 使用 Wrapper 执行构建
if [ -f "gradlew" ]; then
    GRADLE_CMD="./gradlew"
elif [ -f "gradlew.bat" ]; then
    GRADLE_CMD="./gradlew.bat"
else
    GRADLE_CMD="gradle"
fi

# 执行构建
echo "开始构建项目..."
$GRADLE_CMD clean build --info --console plain

if [ $? -eq 0 ]; then
    echo "构建成功！"
else
    echo "构建失败！"
    exit 1
fi

bash
#!/bin/bash
# Gradle 部署脚本示例

PROJECT_VERSION=$(./gradlew properties -q | grep "^version:" | awk '{print $2}')

echo "项目版本: $PROJECT_VERSION"

# 运行测试
./gradlew test

if [ $? -ne 0 ]; then
    echo "测试失败，停止部署"
    exit 1
fi

# 构建发布包
./gradlew build -Prelease=true

# 检查构建产物
if [ -f "build/libs/${PWD##*/}-${PROJECT_VERSION}.jar" ]; then
    echo "构建产物已生成: build/libs/${PWD##*/}-${PROJECT_VERSION}.jar"
    echo "部署准备就绪"
else
    echo "构建产物未找到"
    exit 1
fi

最佳实践 ​

  1. 使用 Gradle Wrapper：确保团队使用相同版本的 Gradle
  2. 启用并行构建：利用多核 CPU 加速构建过程
  3. 使用构建缓存：减少重复构建时间
  4. 合理配置依赖：使用正确的依赖配置（implementation, api, compileOnly 等）
  5. 模块化项目结构：将大型项目拆分为多个模块
  6. 配置文件分离：将不同环境的配置分离到不同文件
  7. 使用最新稳定版本：定期更新 Gradle 和依赖库
  8. 启用增量构建：只构建发生变化的部分
  9. 代码质量检查：集成静态分析和测试工具
  10. 文档化构建过程：为团队成员提供清晰的构建说明
