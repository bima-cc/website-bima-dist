简介 ​

RabbitMQ 是一个开源的消息代理和队列服务器，基于 AMQP（Advanced Message Queuing Protocol）协议实现。它提供了可靠的消息传递、灵活的路由、事务支持和多种消息模式。RabbitMQ 基于 Erlang 语言开发，利用了 Erlang 在可靠性、容错性和分布式方面的优势。

RabbitMQ 的核心概念包括：

  • Producer：消息生产者，负责发送消息到交换器
  • Consumer：消息消费者，从队列中接收消息
  • Exchange：交换器，决定消息如何路由到队列
  • Queue：消息队列，存储待处理的消息
  • Binding：绑定，定义了交换器和队列之间的关系
  • Virtual Host：虚拟主机，用于逻辑隔离

RabbitMQ 广泛应用于异步处理、应用解耦、流量削峰、分布式事务、事件驱动架构等场景。

Docker 方式安装 ​

bash
# 运行 RabbitMQ 容器
docker run -d --name rabbitmq -p 5672:5672 -p 15672:15672 rabbitmq:management

# 运行带持久化存储的 RabbitMQ
docker run -d --name rabbitmq -p 5672:5672 -p 15672:15672 \
  -v rabbitmq-data:/var/lib/rabbitmq \
  rabbitmq:management

# 使用环境变量配置
docker run -d --name rabbitmq -p 5672:5672 -p 15672:15672 \
  -e RABBITMQ_DEFAULT_USER=admin \
  -e RABBITMQ_DEFAULT_PASS=password \
  rabbitmq:management

# 使用 docker-compose 运行
cat <<EOF > docker-compose.yml
version: '3.8'
services:
  rabbitmq:
    image: rabbitmq:3.11-management
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      RABBITMQ_DEFAULT_USER: admin
      RABBITMQ_DEFAULT_PASS: password
    volumes:
      - rabbitmq_data:/var/lib/rabbitmq
      - ./rabbitmq.config:/etc/rabbitmq/rabbitmq.config

volumes:
  rabbitmq_data:
EOF

# 启动服务
docker-compose up -d

基本配置 ​

RabbitMQ 配置文件 (rabbitmq.conf) ​

ini
# 网络配置
listeners.tcp.default = 5672

# 管理插件配置
management.tcp.port = 15672
management.tcp.ip = 0.0.0.0

# 内存配置
total_memory_available_override_value = 2GB

# 磁盘配置
disk_free_limit.relative = 2.0

# 日志配置
log.file.level = info

# 集群配置
cluster_formation.peer_discovery_backend = classic_config
cluster_nodes = [rabbit@node1, rabbit@node2, rabbit@node3]

# 默认用户配置
default_user = guest
default_pass = guest

队列和交换器管理命令 ​

bash

RabbitMQ 管理工具命令 ​

rabbitmqadmin 工具 ​

bash
# 下载 rabbitmqadmin 工具
curl -O http://localhost:15672/cli/rabbitmqadmin
chmod +x rabbitmqadmin

# 列出队列
./rabbitmqadmin list queues

# 列出交换器
./rabbitmqadmin list exchanges

# 创建队列
./rabbitmqadmin declare queue name=myqueue durable=true

# 创建交换器
./rabbitmqadmin declare exchange name=myexchange type=direct

# 创建绑定
./rabbitmqadmin declare binding source=myexchange destination=myqueue routing_key=mykey

# 发布消息
./rabbitmqadmin publish exchange=amq.default routing_key=myqueue payload="Hello, World!"

# 获取消息
./rabbitmqadmin get queue=myqueue ackmode=ack_requeue_false

最佳实践 ​

  1. 安全性：禁用默认 guest 用户，使用强密码，配置 SSL 加密
  2. 监控：监控队列长度、消费者状态、连接数等关键指标
  3. 高可用性：配置镜像队列，使用集群部署
  4. 性能调优：合理配置内存水位线、文件句柄限制等参数
  5. 消息可靠性：启用持久化，使用确认机制
  6. 资源管理：设置适当的队列长度限制，避免资源耗尽
  7. 备份策略：定期备份配置和重要数据
