简介 ​

Grafana 是一个开源的度量分析和可视化套件，主要用于对来自各种数据源的时间序列数据进行查询、可视化、警报和分析。Grafana 支持多种数据源，包括 Prometheus、InfluxDB、Elasticsearch、PostgreSQL、MySQL 等，是现代监控堆栈中不可或缺的组件。

Grafana 的核心功能包括：

  • 仪表板：创建丰富的可视化仪表板
  • 数据源：支持多种时间序列数据库
  • 面板：提供多种图表类型（线图、柱状图、饼图等）
  • 报警：基于查询结果的报警机制
  • 变量：动态仪表板变量
  • 模板：可重用的仪表板模板

Grafana 广泛应用于系统监控、应用性能监控、业务指标跟踪、数据分析等场景。

# 验证安装

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 添加 Grafana APT GPG 密钥
wget -q -O - https://packages.grafana.com/gpg.key | sudo apt-key add -

# 添加稳定版本仓库
echo "deb https://packages.grafana.com/oss/deb stable main" | sudo tee -a /etc/apt/sources.list.d/grafana.list

# 更新包索引
sudo apt update

# 安装 Grafana
sudo apt install grafana

# 启动 Grafana 服务
sudo systemctl daemon-reload
sudo systemctl start grafana-server
sudo systemctl enable grafana-server

# 检查服务状态
sudo systemctl status grafana-server

CentOS/RHEL/Fedora 系统 ​

bash
# 添加 Grafana YUM 仓库
sudo tee /etc/yum.repos.d/grafana.repo << EOF
[grafana]
name=grafana
baseurl=https://packages.grafana.com/oss/rpm
repo_gpgcheck=1
enabled=1
gpgcheck=1
gpgkey=https://packages.grafana.com/gpg.key
sslverify=1
sslcacert=/etc/pki/tls/certs/ca-bundle.crt
EOF

# 安装 Grafana
sudo yum install grafana

# 或者对于较新版本的 RHEL/Fedora
sudo dnf install grafana

# 启动 Grafana 服务
sudo systemctl daemon-reload
sudo systemctl start grafana-server
sudo systemctl enable grafana-server

# 检查服务状态
sudo systemctl status grafana-server

Docker 方式安装 ​

bash
# 运行 Grafana 容器
docker run -d -p 3000:3000 --name=grafana grafana/grafana

# 运行带持久化存储的 Grafana
docker run -d -p 3000:3000 --name=grafana -v grafana-storage:/var/lib/grafana grafana/grafana

# 运行带环境变量配置的 Grafana
docker run -d -p 3000:3000 \
  -e "GF_SECURITY_ADMIN_PASSWORD=mypassword" \
  --name=grafana grafana/grafana

# 使用 docker-compose 运行
cat <<EOF > docker-compose.yml
version: '3.8'
services:
  grafana:
    image: grafana/grafana
    ports:
      - "3000:3000"
    volumes:
      - grafana-storage:/var/lib/grafana
      - ./plugins:/var/lib/grafana/plugins
      - ./conf/provisioning:/etc/grafana/provisioning
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin
      - GF_USERS_ALLOW_SIGN_UP=false
volumes:
  grafana-storage:
EOF

# 启动服务
docker-compose up -d

配置文件 ​

主配置文件 (grafana.ini) ​

ini
[server]
# HTTP 端口
http_port = 3000

# HTTP 地址
http_addr = 0.0.0.0

# 根路径
root_url = http://localhost:3000

[database]
# 数据库类型 (defaults to sqlite3)
type = sqlite3
# 路径 (only applicable to SQLite)
path = grafana.db

[security]
# 管理员密码
admin_user = admin
admin_password = admin

# 匿名访问
# allow_anonymous = true
# anonymous_org_role = Viewer

[users]
# 允许用户注册
allow_sign_up = false

[auth.anonymous]
# 启用匿名访问
enabled = false

[analytics]
# Google Analytics ID (leave empty to disable)
google_analytics_ua_id =

[log]
# 日志级别: debug, info, warn, error, critical
level = info

[paths]
# 数据存储路径
data = /var/lib/grafana
# 临时文件路径
temp_data_lifetime = 24h

Grafana CLI 命令 ​

bash
# 显示 Grafana 版本
grafana-cli --version

# 列出已安装的插件
grafana-cli plugins ls

# 安装插件
grafana-cli plugins install PLUGIN_ID

# 示例：安装 Pie Chart 插件
grafana-cli plugins install grafana-piechart-panel

# 升级插件
grafana-cli plugins upgrade PLUGIN_ID

# 升级所有插件
grafana-cli plugins upgrade-all

# 删除插件
grafana-cli plugins remove PLUGIN_ID

# 清理插件缓存
grafana-cli admin reset-admin-password newpassword

# 创建管理员用户
grafana-cli admin create-admin

组织和用户管理 ​

bash
# 获取所有组织
curl -H "Authorization: Bearer YOUR_API_KEY" \
  http://localhost:3000/api/orgs

# 获取特定组织信息
curl -H "Authorization: Bearer YOUR_API_KEY" \
  http://localhost:3000/api/org

# 获取组织中的用户
curl -H "Authorization: Bearer YOUR_API_KEY" \
  http://localhost:3000/api/org/users

# 添加用户到组织
curl -X POST -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"loginOrEmail":"user@example.com","role":"Editor"}' \
  http://localhost:3000/api/org/users

# 更新用户角色
curl -X PATCH -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"role":"Admin"}' \
  http://localhost:3000/api/org/users/ID

# 删除组织中的用户
curl -X DELETE -H "Authorization: Bearer YOUR_API_KEY" \
  http://localhost:3000/api/org/users/ID

备份和恢复 ​

备份 Grafana 数据 ​

bash
# 备份数据库文件
sudo systemctl stop grafana-server
sudo cp /var/lib/grafana/grafana.db /backup/grafana.db.backup
sudo systemctl start grafana-server

# 备份配置文件
sudo cp /etc/grafana/grafana.ini /backup/grafana.ini.backup

# 备份插件目录
sudo tar -czf /backup/grafana-plugins.tar.gz /var/lib/grafana/plugins

最佳实践 ​

  1. 安全性：更改默认密码，使用 HTTPS，限制访问权限
  2. 性能：合理配置缓存，优化查询，使用适当的硬件资源
  3. 备份：定期备份数据库和配置文件
  4. 监控：监控 Grafana 自身的性能指标
  5. 插件管理：定期更新插件，只安装必需的插件
  6. 用户管理：使用适当的角色和权限控制
  7. 仪表板设计：创建清晰、易读的仪表板，使用合适的图表类型
