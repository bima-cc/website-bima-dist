常用 Nginx 命令技术文档 ​

简介 ​

Nginx（发音为 "engine-x"）是一个高性能的 HTTP 和反向代理 web 服务器，同时也提供了 IMAP/POP3/SMTP 代理服务。Nginx 以其高并发、低内存消耗和稳定性而闻名，被广泛应用于静态内容服务、负载均衡、反向代理和 SSL 终端等领域。

Nginx 的主要特性包括：

  • 高并发处理能力（基于事件驱动的异步架构）
  • 低内存消耗
  • 高稳定性
  • 热部署支持
  • 模块化设计
  • 强大的负载均衡能力

Nginx 广泛应用于网站托管、API 网关、微服务架构、CDN 边缘服务器等场景。

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash

sudo systemctl start nginx

# 设置开机自启
sudo systemctl enable nginx

# 检查 Nginx 状态
sudo systemctl status nginx

# 验证安装
nginx -v

  2. 下载 Windows 版本的 Nginx
  3. 解压到目标目录
  4. 在命令提示符中运行 nginx.exe

语法 ​

Nginx 命令的基本语法结构：

nginx [-?hvVtTq] [-s signal] [-p prefix] [-c conf_file] [-g directives]
1

常用选项：

  • -?, -h: 显示帮助信息
  • -v: 显示版本信息
  • -V: 显示版本和配置参数
  • -t: 测试配置文件语法
  • -T: 测试配置文件并转储有效配置
  • -q: 在测试期间抑制非错误消息
  • -s signal: 发送信号到主进程
  • -p prefix: 设置前缀路径
  • -c conf_file: 使用替代配置文件
  • -g directives: 设置全局配置指令

bash

sudo nginx

sudo nginx -T

# 查看 Nginx 进程
ps aux | grep nginx

# 检查 Nginx 服务状态
sudo systemctl status nginx

# 重启 Nginx 服务
sudo systemctl restart nginx

# 重新加载 Nginx 服务
sudo systemctl reload nginx

配置文件管理 ​

bash
# 查看主配置文件
sudo cat /etc/nginx/nginx.conf

# 编辑主配置文件
sudo nano /etc/nginx/nginx.conf

# 检查配置文件语法
sudo nginx -t

sudo nano /etc/nginx/sites-available/site_name

sudo ln -s /etc/nginx/sites-available/site_name /etc/nginx/sites-enabled/

sudo rm /etc/nginx/sites-enabled/site_name

# 创建 SSL 配置模板
sudo nano /etc/nginx/snippets/ssl-params.conf

基本配置示例 ​

nginx
# /etc/nginx/nginx.conf
user www-data;
worker_processes auto;
pid /run/nginx.pid;
include /etc/nginx/modules-enabled/*.conf;

events {
    worker_connections 768;
}

http {
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;
    
    include /etc/nginx/mime.types;
    default_type application/octet-stream;
    
    # 日志格式
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';
    
    access_log /var/log/nginx/access.log main;
    error_log /var/log/nginx/error.log;
    
    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    
    # 包含站点配置
    include /etc/nginx/conf.d/*.conf;
    include /etc/nginx/sites-enabled/*;
}

虚拟主机配置 ​

nginx
# /etc/nginx/sites-available/example.com
server {
    listen 80;
    listen [::]:80;
    server_name example.com www.example.com;
    
    # 网站根目录
    root /var/www/example.com/html;
    index index.html index.htm index.nginx-debian.html;
    
    # 访问日志
    access_log /var/log/nginx/example.com.access.log;
    error_log /var/log/nginx/example.com.error.log;
    
    # 默认位置块
    location / {
        try_files $uri $uri/ =404;
    }
    
    # 错误页面
    error_page 404 /404.html;
    error_page 500 502 503 504 /50x.html;
    
    location = /50x.html {
        root /var/www/html;
    }
}

# SSL 配置示例
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name example.com www.example.com;
    
    ssl_certificate /path/to/certificate.crt;
    ssl_certificate_key /path/to/private.key;
    
    # SSL 安全配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512:ECDHE-RSA-AES256-GCM-SHA384:DHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;
    
    root /var/www/example.com/html;
    index index.html index.htm index.php;
    
    location / {
        try_files $uri $uri/ =404;
    }
}

反向代理配置 ​

nginx
# 基本反向代理配置
server {
    listen 80;
    server_name proxy.example.com;
    
    location / {
        proxy_pass http://backend_server;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}

# 负载均衡配置
upstream backend_servers {
    server 192.168.1.10:8080 weight=3;
    server 192.168.1.11:8080 weight=2;
    server 192.168.1.12:8080 backup;
}

server {
    listen 80;
    server_name loadbalancer.example.com;
    
    location / {
        proxy_pass http://backend_servers;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

PHP-FPM 集成 ​

nginx
# PHP 应用配置
server {
    listen 80;
    server_name phpapp.example.com;
    root /var/www/phpapp;
    index index.php index.html;
    
    # 静态文件处理
    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # PHP 文件处理
    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
        # 或者使用 TCP 连接
        # fastcgi_pass 127.0.0.1:9000;
    }
    
    # 隐藏 .htaccess 文件
    location ~ /\.ht {
        deny all;
    }
}

安全配置 ​

nginx
# 安全头配置
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header X-Content-Type-Options "nosniff" always;
add_header Referrer-Policy "no-referrer-when-downgrade" always;
add_header Content-Security-Policy "default-src 'self' http: https: data: blob: 'unsafe-inline'" always;

# 隐藏版本信息
server_tokens off;

# 限制请求大小
client_max_body_size 10M;

# 限制请求频率
limit_req_zone $binary_remote_addr zone=login:10m rate=1r/s;
location /login {
    limit_req zone=login burst=5 nodelay;
}

# 阻止特定 IP
deny 192.168.1.100;
allow 192.168.1.0/24;
deny all;

日志管理 ​

bash
# 查看访问日志
sudo tail -f /var/log/nginx/access.log

# 查看错误日志
sudo tail -f /var/log/nginx/error.log

# 按日期分割日志
sudo logrotate /etc/nginx/nginx.conf

# 统计访问量
sudo awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -nr

# 查找 404 错误
sudo awk '$9 == 404 {print $7}' /var/log/nginx/access.log | sort | uniq -c | sort -nr

# 统计访问最多的 URL
sudo awk '{print $7}' /var/log/nginx/access.log | sort | uniq -c | sort -nr | head -20

性能优化 ​

nginx
# /etc/nginx/nginx.conf 优化配置
worker_processes auto;
worker_cpu_affinity auto;

events {
    worker_connections 1024;
    use epoll;
    multi_accept on;
}

http {
    # 优化连接处理
    keepalive_timeout 65;
    keepalive_requests 100;
    
    # 优化缓冲区
    client_body_buffer_size 128k;
    client_max_body_size 10m;
    client_header_buffer_size 1k;
    large_client_header_buffers 4 4k;
    output_buffers 1 32k;
    postpone_output 1460;
    
    # 优化文件传输
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    
    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_comp_level 6;
    gzip_types
        text/plain
        text/css
        text/xml
        text/javascript
        application/javascript
        application/xml+rss
        application/json;
}

常用配置片段 ​

bash
# 创建 SSL 参数配置
sudo mkdir -p /etc/nginx/snippets
sudo tee /etc/nginx/snippets/ssl-params.conf << EOF
ssl_protocols TLSv1.2 TLSv1.3;
ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512:ECDHE-RSA-AES256-GCM-SHA384:DHE-RSA-AES256-GCM-SHA384;
ssl_prefer_server_ciphers off;
ssl_session_cache shared:SSL:10m;
ssl_session_timeout 10m;
add_header X-Frame-Options DENY;
add_header X-Content-Type-Options nosniff;
add_header Strict-Transport-Security "max-age=63072000; includeSubDomains; preload";
EOF

最佳实践 ​

  1. 安全配置：始终隐藏版本号，设置适当的安全头
  2. SSL/TLS：使用最新的 TLS 版本和强加密套件
  3. 性能调优：根据服务器硬件调整 worker_processes 和 worker_connections
  4. 日志管理：合理配置日志级别，定期轮转日志文件
  5. 备份配置：定期备份配置文件，使用版本控制系统
  6. 测试配置：每次修改后使用 nginx -t 测试配置文件
  7. 资源限制：设置适当的客户端请求大小和连接数限制
  8. 缓存策略：为静态资源设置适当的缓存头
  9. 监控告警：监控 Nginx 性能指标和错误日志
  10. 定期更新：保持 Nginx 版本更新以获得安全补丁
