常用 Jenkins 命令技术文档 ​

简介 ​

Jenkins 是一个开源的自动化服务器，用于自动化各种与构建、测试和部署软件相关的任务。Jenkins 提供了数百个插件，支持构建、部署和自动化软件开发流程，是 DevOps 工具链中的核心组件之一。

Jenkins 的主要特点包括：

  • 基于插件的架构，具有高度可扩展性
  • 支持持续集成和持续部署（CI/CD）
  • 提供友好的 Web 界面进行配置和管理
  • 支持分布式构建
  • 丰富的插件生态系统
  • 支持流水线即代码（Pipeline as Code）
  • 跨平台支持

Jenkins 广泛应用于软件开发的自动化构建、测试、部署流程中，是现代 DevOps 实践中不可或缺的工具。

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 更新包索引
sudo apt update

# 安装 Java（Jenkins 依赖 Java 运行时）
sudo apt install openjdk-11-jdk

# 添加 Jenkins APT 密钥
curl -fsSL https://pkg.jenkins.io/debian-stable/jenkins.io-2023.key | sudo tee \
  /usr/share/keyrings/jenkins-keyring.asc > /dev/null

# 添加 Jenkins APT 仓库
echo deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc] \
  https://pkg.jenkins.io/debian-stable binary/ | sudo tee \
  /etc/apt/sources.list.d/jenkins.list > /dev/null

# 更新包索引
sudo apt update

# 安装 Jenkins
sudo apt install jenkins

# 启动 Jenkins 服务
sudo systemctl start jenkins

# 设置开机自启
sudo systemctl enable jenkins

# 检查 Jenkins 服务状态
sudo systemctl status jenkins

# 验证安装
sudo systemctl status jenkins

CentOS/RHEL/Fedora 系统 ​

bash
# 安装 Java
sudo yum install java-11-openjdk-devel    # CentOS/RHEL 7
sudo dnf install java-11-openjdk-devel    # Fedora/CentOS Stream

# 添加 Jenkins YUM 密钥

# 添加 Jenkins YUM 仓库
sudo tee /etc/yum.repos.d/jenkins.repo<<EOF
[jenkins]
name=Jenkins-stable
baseurl=https://pkg.jenkins.io/redhat-stable
enabled=1
gpgcheck=1
EOF

# 安装 Jenkins
sudo yum install jenkins    # CentOS/RHEL 7
sudo dnf install jenkins    # Fedora/CentOS Stream

# 启动 Jenkins 服务
sudo systemctl start jenkins

# 设置开机自启
sudo systemctl enable jenkins

# 检查 Jenkins 服务状态
sudo systemctl status jenkins

作业管理 ​

bash
# 使用 Jenkins API 创建作业
curl -X POST -H "Content-Type: application/xml" -u username:token \
  --data-binary @job-config.xml \
  http://jenkins-server:8080/createItem?name=job-name

# 获取作业信息
curl -u username:token http://jenkins-server:8080/job/job-name/api/json

# 触发构建
curl -X POST -u username:token http://jenkins-server:8080/job/job-name/build

# 带参数的构建

  "http://jenkins-server:8080/job/job-name/buildWithParameters?param1=value1&param2=value2"

# 停止构建
curl -X POST -u username:token http://jenkins-server:8080/job/job-name/1/stop

流水线（Pipeline）操作 ​

bash
# 创建流水线作业配置文件示例
cat > pipeline-job.xml << 'EOF'
<?xml version='1.1' encoding='UTF-8'?>
<flow-definition plugin="workflow-job@1143.v8a_3fb_da_341e">
  <actions/>
  <description>Sample Pipeline Job</description>
  <keepDependencies>false</keepDependencies>
  <properties>
    <jenkins.model.BuildDiscarderProperty>
      <strategy class="hudson.tasks.LogRotator">
        <daysToKeep>-1</daysToKeep>
        <numToKeep>10</numToKeep>
        <artifactDaysToKeep>-1</artifactDaysToKeep>
        <artifactNumToKeep>-1</artifactNumToKeep>
      </strategy>
    </jenkins.model.BuildDiscarderProperty>
  </properties>
  <definition class="org.jenkinsci.plugins.workflow.cps.CpsFlowDefinition" plugin="workflow-cps@2648.va_9a_a_1663d1e">
    <script>
pipeline {
    agent any
    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }
        stage('Build') {
            steps {
                sh 'echo "Building..."'
            }
        }
        stage('Test') {
            steps {
                sh 'echo "Testing..."'
            }
        }
        stage('Deploy') {
            steps {
                sh 'echo "Deploying..."'
            }
        }
    }
}
    </script>
    <sandbox>true</sandbox>
  </definition>
  <triggers/>
  <disabled>false</disabled>
</flow-definition>
EOF

# 使用配置文件创建流水线作业
java -jar jenkins-cli.jar -s http://jenkins-server:8080/ create-job pipeline-job < pipeline-job.xml

bash

sudo rm -rf /var/lib/jenkins/*
sudo tar -xzf jenkins-backup-*.tar.gz -C /
sudo chown -R jenkins:jenkins /var/lib/jenkins
sudo systemctl start jenkins

自动化脚本 ​

bash
#!/bin/bash
# 批量创建 Jenkins 作业的脚本

JENKINS_URL="http://localhost:8080"
JENKINS_USER="admin"
JENKINS_TOKEN="your-token"
JOBS=("job1" "job2" "job3")

for job in "${JOBS[@]}"; do
  echo "Creating job: $job"
  curl -X POST -H "Content-Type: application/xml" \
    -u "$JENKINS_USER:$JENKINS_TOKEN" \
    --data-binary @job-template.xml \
    "$JENKINS_URL/createItem?name=$job"
done

最佳实践 ​

  1. 安全配置：启用适当的身份验证和授权机制，定期更新凭据
  2. 备份策略：定期备份 Jenkins 配置和作业数据
  3. 插件管理：定期更新插件，移除不需要的插件
  4. 权限控制：实施最小权限原则，限制用户访问权限
  5. 流水线即代码：使用 Jenkinsfile 定义构建流程
  6. 分布式构建：使用构建节点分散负载
  7. 监控告警：监控 Jenkins 性能和构建状态
  8. 版本控制：将 Jenkins 配置存储在版本控制系统中
  9. 定期维护：定期清理旧的构建数据，优化性能
  10. 文档记录：记录 Jenkins 配置和维护流程
  11. 环境管理：为不同环境（开发、测试、生产）配置不同的节点
  12. 通知机制：配置适当的构建结果通知
