常用 Docker Compose 命令技术文档 ​

简介 ​

Docker Compose 是 Docker 官方提供的容器编排工具，用于定义和运行多容器 Docker 应用程序。通过一个 YAML 文件（docker-compose.yml）来配置应用程序的服务，使用一个命令即可创建并启动所有配置的服务。Docker Compose 简化了多容器应用的部署和管理，是开发、测试和小型部署环境的理想选择。

Docker Compose 的核心概念包括：

  • Services：定义应用程序的各个服务组件
  • Networks：定义服务间的网络连接
  • Volumes：定义数据卷用于持久化存储
  • Configs：定义配置文件
  • Secrets：定义敏感信息

Docker Compose 广泛应用于本地开发环境、测试环境搭建、微服务应用部署、CI/CD 流程等场景。

基本配置 ​

docker-compose.yml 示例配置 ​

yaml
version: '3.8'

services:
  # Web 应用服务
  web:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - .:/code
    environment:
      - DEBUG=1
    depends_on:
      - redis
      - db
    networks:
      - frontend
      - backend

  # 数据库服务
  db:
    image: postgres:13
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      POSTGRES_DB: myapp
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: password
    networks:
      - backend

  # 缓存服务
  redis:
    image: redis:6-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    networks:
      - backend

  # 反向代理
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
    depends_on:
      - web
    networks:
      - frontend

volumes:
  postgres_data:
  redis_data:

networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge

高级配置示例 ​

yaml
version: '3.8'

services:
  app:
    image: myapp:latest
    deploy:
      replicas: 3
      restart_policy:
        condition: on-failure
        delay: 5s
        max_attempts: 3
      resources:
        limits:
          cpus: '0.5'
          memory: 512M
        reservations:
          cpus: '0.25'
          memory: 256M
    environment:
      - ENV=production
    configs:
      - source: app_config
        target: /etc/myapp/config.yaml
    secrets:
      - db_password
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

configs:
  app_config:
    file: ./configs/app.yaml

secrets:
  db_password:
    file: ./secrets/db_password.txt

项目生命周期管理 ​

启动和停止服务 ​

bash
# 构建、创建并启动所有服务
docker-compose up

# 后台运行（分离模式）
docker-compose up -d

# 构建镜像并启动（如果有变化）
docker-compose up --build

# 重建并启动特定服务
docker-compose up --build service_name

# 停止并删除容器
docker-compose down

# 停止并删除容器、网络和镜像
docker-compose down --rmi all

# 停止并删除容器、网络和挂载卷
docker-compose down -v

# 停止所有服务（不删除容器）
docker-compose stop

# 启动已停止的服务
docker-compose start

# 重启服务
docker-compose restart

常用脚本示例 ​

bash
#!/bin/bash
# docker-compose-helpful.sh

# 项目启动脚本
start_project() {
    echo "Starting project..."
    docker-compose up -d --build
    docker-compose logs -f
}

# 项目停止脚本
stop_project() {
    echo "Stopping project..."
    docker-compose down
}

# 数据库备份脚本
backup_db() {
    TIMESTAMP=$(date +"%Y%m%d_%H%M%S")
    docker-compose exec db pg_dump -U postgres myapp > "backup_${TIMESTAMP}.sql"
    echo "Backup created: backup_${TIMESTAMP}.sql"
}

# 根据参数执行对应函数
case $1 in
    start)
        start_project
        ;;
    stop)
        stop_project
        ;;
    backup)
        backup_db
        ;;
    *)
        echo "Usage: $0 {start|stop|backup}"
        exit 1
        ;;
esac

最佳实践 ​

  1. 版本控制：将 docker-compose.yml 文件纳入版本控制
  2. 环境分离：使用不同的 compose 文件管理不同环境
  3. 资源限制：为容器设置适当的 CPU 和内存限制
  4. 健康检查：配置服务健康检查以确保依赖关系正确
  5. 日志管理：配置适当的日志驱动和日志轮转
  6. 安全性：使用 secrets 管理敏感信息，避免在配置中硬编码
  7. 网络隔离：合理设计网络以隔离不同服务
  8. 备份策略：制定数据卷的备份和恢复策略
