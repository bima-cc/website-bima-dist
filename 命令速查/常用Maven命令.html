常用 Maven 命令技术文档 ​

简介 ​

Apache Maven 是一个强大的项目管理和理解工具，主要用于 Java 项目的构建、依赖管理和项目信息管理。Maven 基于项目对象模型（POM）的概念，通过一小段描述信息来管理项目的构建、报告和文档。它提供了一种标准化的项目结构和一套通用的构建生命周期，大大简化了 Java 项目的构建过程。

Maven 的主要特点包括：

  • 依赖管理：自动下载和管理项目依赖
  • 标准化项目结构：统一的目录结构和构建流程
  • 构建生命周期：预定义的构建阶段和目标
  • 插件系统：丰富的插件生态系统
  • 项目继承：支持多模块项目和继承
  • 仓库管理：中央仓库和私有仓库支持
  • 报告生成：自动生成项目报告和文档

Maven 广泛应用于 Java 企业级应用开发、Spring Boot 项目、微服务架构以及各类 Java 开源项目中，是 Java 生态系统中最重要和最常用的构建工具之一。

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 使用 apt 安装（包含 OpenJDK）
sudo apt update
sudo apt install maven

# 验证安装
mvn --version

# 或者手动安装
# 1. 下载 Maven
cd /tmp
wget https://archive.apache.org/dist/maven/maven-3/3.9.6/binaries/apache-maven-3.9.6-bin.tar.gz

# 2. 解压到目标目录
sudo tar -xzf apache-maven-3.9.6-bin.tar.gz -C /opt
sudo mv /opt/apache-maven-3.9.6 /opt/maven

# 3. 设置环境变量
echo 'export M2_HOME=/opt/maven' >> ~/.bashrc
echo 'export PATH=$PATH:$M2_HOME/bin' >> ~/.bashrc
source ~/.bashrc

# 4. 验证安装
mvn --version

语法 ​

Maven 命令的基本语法结构：

mvn [plugin:goal] [phase] [options]
1

常用选项：

  • -D, --define <property=value>: 定义系统属性
  • -P, --activate-profiles <profile-id>: 激活指定的 profile
  • -X, --debug: 启用调试输出
  • -e, --errors: 启用详细错误信息
  • -q, --quiet: 安静模式
  • -B, --batch-mode: 批处理模式（非交互式）
  • -U, --update-snapshots: 强制更新 snapshots 和 releases
  • -T, --threads <thread-count>: 并行构建线程数
  • -f, --file <file>: 指定 pom.xml 文件位置
  • -s, --settings <file>: 指定 settings.xml 文件位置
  • -o, --offline: 离线模式运行

if [ $? -eq 0 ]; then
    echo "构建成功！"
else
    echo "构建失败！"
    exit 1
fi

# 运行测试
echo "运行测试..."
mvn test

if [ $? -eq 0 ]; then
    echo "测试通过！"
else
    echo "测试失败！"
    exit 1
fi

#!/bin/bash
# Maven 部署脚本示例

# 设置项目版本
VERSION=$(mvn help:evaluate -Dexpression=project.version -q -DforceStdout)

echo "项目版本: $VERSION"

# 检查是否有未提交的更改
if [[ -n $(git status --porcelain) ]]; then
    echo "警告: 存在未提交的更改"
    read -p "是否继续? (y/n): " -n 1 -r
    echo
    if [[ ! $REPLY =~ ^[Yy]$ ]]; then
        exit 1
    fi
fi

# 执行完整构建流程
mvn clean verify -P release

if [ $? -ne 0 ]; then
    echo "构建失败，停止部署"
    exit 1
fi

# 检查构建产物
if [ -f "target/$(basename $(pwd))-$VERSION.jar" ]; then
    echo "构建产物已生成"
    echo "部署准备就绪"
else
    echo "构建产物未找到"
    exit 1
fi

最佳实践 ​

  1. 使用 Maven Wrapper：确保团队使用相同版本的 Maven
  2. 合理配置依赖范围：正确使用 compile, provided, runtime, test, system 范围
  3. 模块化项目结构：将大型项目拆分为多个模块
  4. 使用 BOM 管理依赖版本：通过 Bill of Materials 统一管理版本
  5. 配置文件分离：使用 Profiles 管理不同环境的配置
  6. 启用并行构建：利用多核 CPU 加速构建过程
  7. 定期更新依赖：保持依赖库的最新安全版本
  8. 使用合适的仓库镜像：配置国内镜像加速下载
  9. 代码质量检查：集成静态分析和安全扫描工具
  10. 文档化构建过程：为团队成员提供清晰的构建说明
