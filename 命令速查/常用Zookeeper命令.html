常用 Zookeeper 命令技术文档 ​

简介 ​

Apache ZooKeeper 是一个开源的分布式协调服务，主要用于维护配置信息、命名、提供分布式同步和组服务。ZooKeeper 提供了一个简单的原语集，分布式应用程序可以基于这些原语实现更高级别的服务，如配置维护、命名服务、分布式同步、集群管理等。

ZooKeeper 的主要特点包括：

  • 高可用性：通过集群方式提供服务，即使部分节点失效也能正常工作
  • 一致性：保证客户端视图的一致性
  • 顺序性：对更新请求的处理顺序与它们被接收的顺序一致
  • 简单性：提供一个精简的服务原语集
  • 可靠性：一旦一个消息被交付，它将被持久保存直到被覆盖

ZooKeeper 广泛应用于 Hadoop 生态系统、Kafka、HBase、Solr 等分布式系统中，用于协调和管理分布式应用的状态。

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 安装 Java（ZooKeeper 依赖 Java 运行时）
sudo apt update
sudo apt install openjdk-11-jdk

# 下载 ZooKeeper
cd /tmp
wget https://archive.apache.org/dist/zookeeper/zookeeper-3.8.3/apache-zookeeper-3.8.3-bin.tar.gz

# 解压安装包
tar -xzf apache-zookeeper-3.8.3-bin.tar.gz

# 移动到安装目录
sudo mv apache-zookeeper-3.8.3-bin /opt/zookeeper
sudo chown -R $USER:$USER /opt/zookeeper

# 创建数据目录
sudo mkdir -p /var/lib/zookeeper
sudo chown $USER:$USER /var/lib/zookeeper

# 创建日志目录
sudo mkdir -p /var/log/zookeeper
sudo chown $USER:$USER /var/log/zookeeper

  --name zookeeper \

语法 ​

ZooKeeper 命令的基本语法结构：

zkCli.sh -server host:port [options]
1

常用选项：

  • -server host:port: 指定 ZooKeeper 服务器地址和端口
  • -timeout timeout: 设置会话超时时间（毫秒）
  • -r: 连接到只读模式的服务器
  • -help: 显示帮助信息

ZooKeeper 内部命令语法：

  • command [path] [data] [options]

客户端连接 ​

bash
# 连接到本地 ZooKeeper 服务器
zkCli.sh -server localhost:2181

# 连接到远程 ZooKeeper 服务器
zkCli.sh -server hostname:2181

# 连接到多个 ZooKeeper 服务器（集群）
zkCli.sh -server host1:2181,host2:2181,host3:2181

# 设置连接超时时间
zkCli.sh -server localhost:2181 -timeout 30000

# 退出客户端
quit

权限控制 ​

bash
# 创建带权限的节点
create /secure_node "secret_data" world:anyone:r world:auth:rwa

# 设置特定用户的权限
addauth digest username:password
create /private_node "private_data" auth:username:crwa

# 使用 Digest 认证
addauth digest user1:password1
setAcl /node_name auth:user1:crwa

# 常用权限标识
# c: CREATE - 创建子节点权限
# r: READ - 读取节点数据和列出子节点权限
# w: WRITE - 设置节点数据权限
# d: DELETE - 删除子节点权限
# a: ADMIN - 设置权限权限

实用脚本示例 ​

bash
#!/bin/bash
# ZooKeeper 状态检查脚本

ZK_HOST=localhost
ZK_PORT=2181

# 检查 ZooKeeper 是否运行
if echo ruok | nc $ZK_HOST $ZK_PORT | grep -q imok; then
    echo "ZooKeeper is running"
    # 获取详细状态
    echo stat | nc $ZK_HOST $ZK_PORT
else
    echo "ZooKeeper is not running"
fi

# 监控 ZooKeeper 性能指标
echo mntr | nc $ZK_HOST $ZK_PORT

最佳实践 ​

  1. 集群部署：生产环境应部署奇数个节点（通常为3或5个）
  2. 硬件配置：使用专用服务器，避免与其他服务共享
  3. 数据目录：将数据目录和日志目录放在不同的物理磁盘上
  4. 网络配置：确保节点间网络延迟低且稳定
  5. 安全配置：启用 ACL 控制，配置防火墙规则
  6. 监控告警：持续监控集群状态和性能指标
  7. 定期维护：清理旧的日志文件，定期备份数据
  8. 调优参数：根据实际负载调整 tickTime、initLimit、syncLimit 等参数
  9. 客户端配置：合理配置连接超时和重试机制
  10. 版本管理：及时升级到稳定版本以获得安全补丁和性能改进
