常用 MongoDB 命令技术文档 ​

简介 ​

MongoDB 是一个基于分布式文件存储的开源 NoSQL 数据库系统，属于文档型数据库。MongoDB 采用 BSON（Binary JSON）格式存储数据，提供高性能、高可用性和易扩展性，被广泛应用于 Web 应用、移动应用、大数据分析和实时分析等场景。

MongoDB 的主要特点包括：

  • 面向文档的数据模型（类似 JSON）
  • 高性能的查询能力
  • 水平扩展能力（分片）
  • 自动故障转移和恢复
  • 丰富的查询语言
  • 支持索引和聚合框架

MongoDB 广泛应用于内容管理系统、物联网、实时分析、移动应用后端和微服务架构等场景。

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 导入 MongoDB 公钥
wget -qO - https://www.mongodb.org/static/pgp/server-6.0.asc | sudo apt-key add -

# 添加 MongoDB 仓库
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu focal/mongodb-org/6.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-6.0.list

# 更新包索引
sudo apt update

# 安装 MongoDB 社区版
sudo apt install mongodb-org

# 启动 MongoDB 服务
sudo systemctl start mongod

# 设置开机自启
sudo systemctl enable mongod

# 检查 MongoDB 服务状态
sudo systemctl status mongod

# 验证安装
mongod --version
mongo --version

语法 ​

MongoDB 命令的基本语法结构：

MongoDB Shell 语法 ​

mongo [options] [connection_string]
1

MongoDB 查询语法 ​

db.collection.operation(query, options)
1

常用选项：

  • --host HOST: 指定主机名
  • --port PORT: 指定端口号
  • --username USERNAME: 指定用户名
  • --password PASSWORD: 指定密码
  • --authenticationDatabase DB: 指定认证数据库
  • --eval JAVASCRIPT: 执行 JavaScript 代码
  • --quiet: 安静模式（不显示启动信息）

文档增删改查（CRUD） ​

javascript
// 插入单个文档
db.users.insertOne({
  name: "John Doe",
  email: "john@example.com",
  age: 30,
  createdAt: new Date()
})

// 插入多个文档
db.users.insertMany([
  {name: "Alice", age: 25},
  {name: "Bob", age: 35},
  {name: "Charlie", age: 45}
])

// 查找所有文档
db.users.find()

// 格式化输出
db.users.find().pretty()

// 查找特定条件的文档
db.users.find({age: {$gte: 30}})
db.users.find({name: "John Doe"})
db.users.find({age: {$in: [25, 30, 35]}})

// 查找单个文档
db.users.findOne({_id: ObjectId("...")})

// 更新单个文档
db.users.updateOne(
  {name: "John Doe"},  // 查询条件
  {$set: {age: 31}}    // 更新操作
)

// 更新多个文档
db.users.updateMany(
  {age: {$lt: 30}},
  {$inc: {age: 1}}
)

// 替换文档
db.users.replaceOne(
  {name: "John Doe"},
  {name: "Jane Doe", age: 28}
)

// 删除单个文档
db.users.deleteOne({name: "John Doe"})

// 删除多个文档
db.users.deleteMany({age: {$lt: 30}})

// 删除所有文档
db.users.deleteMany({})

高级查询操作 ​

javascript
// 投影（只返回特定字段）
db.users.find({}, {name: 1, email: 1})  // 包含指定字段
db.users.find({}, {name: 0})           // 排除指定字段

// 排序
db.users.find().sort({age: 1})     // 升序
db.users.find().sort({age: -1})    // 降序

// 限制结果数量
db.users.find().limit(5)

// 跳过指定数量的文档
db.users.find().skip(10)

// 组合使用
db.users.find().sort({age: -1}).skip(5).limit(10)

// 计数
db.users.count()
db.users.countDocuments({age: {$gte: 30}})

// 条件操作符
db.users.find({age: {$gt: 25}})          // 大于
db.users.find({age: {$gte: 25}})         // 大于等于
db.users.find({age: {$lt: 40}})          // 小于
db.users.find({age: {$lte: 40}})         // 小于等于
db.users.find({age: {$ne: 30}})          // 不等于
db.users.find({age: {$in: [25, 30, 35]}})  // 在列表中
db.users.find({age: {$nin: [25, 30, 35]}}) // 不在列表中

// 逻辑操作符
db.users.find({$and: [{age: {$gte: 25}}, {age: {$lte: 40}}]})
db.users.find({$or: [{age: {$lt: 25}}, {age: {$gt: 40}}]})
db.users.find({age: {$not: {$lt: 25}}})

// 正则表达式
db.users.find({name: /john/i})           // 不区分大小写
db.users.find({name: {$regex: "^J"}})    // 以 J 开头

// 数组查询
db.users.find({"hobbies": "reading"})           // 包含特定值
db.users.find({"hobbies": {$all: ["reading", "swimming"]}})  // 包含所有值
db.users.find({"hobbies": {$size: 3}})         // 数组大小

javascript

聚合操作 ​

javascript
// 简单聚合管道
db.orders.aggregate([
  {$match: {status: "completed"}},
  {$group: {_id: "$customer_id", total: {$sum: "$amount"}}},
  {$sort: {total: -1}},
  {$limit: 10}
])

// 常用聚合操作符
db.users.aggregate([
  {$match: {age: {$gte: 18}}},
  {$group: {
    _id: "$city",
    count: {$sum: 1},
    avgAge: {$avg: "$age"},
    maxAge: {$max: "$age"},
    minAge: {$min: "$age"},
    totalAge: {$sum: "$age"}
  }}
])

// 投影和重命名字段
db.users.aggregate([
  {$project: {
    fullName: {$concat: ["$firstName", " ", "$lastName"]},
    email: 1,
    age: 1,
    _id: 0
  }}
])

备份和恢复 ​

bash

mongodump

# 备份特定数据库
mongodump --db database_name

# 备份特定集合
mongodump --db database_name --collection collection_name

# 恢复备份
mongorestore dump/

# 导出特定集合为 JSON

# 导入 JSON 数据
mongoimport --db database_name --collection collection_name input.json

最佳实践 ​

  1. 设计模式：合理设计文档结构，避免过度嵌套
  2. 索引策略：为常用查询字段创建索引，但避免过多索引影响写入性能
  3. 安全配置：启用认证，限制网络访问，使用安全连接
  4. 备份策略：定期备份数据，测试恢复过程
  5. 监控告警：监控数据库性能指标和资源使用情况
  6. 连接池管理：合理配置应用连接池参数
  7. 查询优化：使用投影减少数据传输，避免全表扫描
  8. 容量规划：根据数据增长趋势规划存储容量
  9. 分片策略：当数据量增长时考虑分片方案
  10. 版本升级：定期升级 MongoDB 版本以获得新功能和安全补丁
