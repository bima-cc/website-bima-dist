常用 Prometheus 命令技术文档 ​

简介 ​

Prometheus 是一个开源的系统监控和报警工具包，最初由 SoundCloud 开发，现由 Cloud Native Computing Foundation（CNCF）维护。Prometheus 采用时间序列数据库存储指标数据，使用 Pull 模型收集数据，具有强大的查询语言 PromQL，广泛应用于微服务和云原生环境的监控。

Prometheus 的核心组件包括：

  • Prometheus Server：负责抓取和存储时间序列数据
  • Exporters：将第三方系统的数据转换为 Prometheus 可理解的格式
  • Alertmanager：处理报警通知
  • Pushgateway：接收短期作业的指标推送

Prometheus 广泛应用于服务监控、应用性能监控、基础设施监控、告警系统等场景。

systemd 服务配置（Linux） ​

bash
# 创建 Prometheus 用户
sudo useradd --no-create-home --shell /bin/false prometheus

# 创建必要目录
sudo mkdir -p /etc/prometheus
sudo mkdir -p /var/lib/prometheus

# 设置权限
sudo chown prometheus:prometheus /etc/prometheus
sudo chown prometheus:prometheus /var/lib/prometheus

# 移动二进制文件到系统路径
sudo cp prometheus /usr/local/bin/
sudo cp promtool /usr/local/bin/

# 设置权限
sudo chown prometheus:prometheus /usr/local/bin/prometheus
sudo chown prometheus:prometheus /usr/local/bin/promtool

# 创建 systemd 服务文件
sudo tee /etc/systemd/system/prometheus.service << EOF
[Unit]
Description=Prometheus
Wants=network-online.target
After=network-online.target

[Service]
User=prometheus
Group=prometheus
Type=simple
ExecStart=/usr/local/bin/prometheus \
    --config.file /etc/prometheus/prometheus.yml \
    --storage.tsdb.path /var/lib/prometheus/ \
    --web.console.templates=/etc/prometheus/consoles \
    --web.console.libraries=/etc/prometheus/console_libraries

[Install]
WantedBy=multi-user.target
EOF

# 重新加载 systemd
sudo systemctl daemon-reload

# 启动 Prometheus
sudo systemctl start prometheus

# 设置开机自启
sudo systemctl enable prometheus

Prometheus Server 启动命令 ​

bash

# 指定数据存储路径
prometheus --config.file=prometheus.yml --storage.tsdb.path=/var/lib/prometheus/

prometheus --config.file=prometheus.yml --web.listen-address="0.0.0.0:9090"

# 启用 Admin API（生产环境需谨慎）
prometheus --config.file=prometheus.yml --web.enable-admin-api

# 指定外部 URL
prometheus --config.file=prometheus.yml --web.external-url="http://prometheus.example.com"

Web 界面操作 ​

Prometheus 表达式浏览器 ​

Prometheus 提供了一个 Web 界面，可以通过 HTTP API 查询数据：

  • 访问地址：http://your-prometheus-host:9090
  • 表达式查询界面：http://your-prometheus-host:9090/graph
  • 直接查询 API：http://your-prometheus-host:9090/api/v1/query

常用 PromQL 查询 ​

promql
# 查看所有活跃时间序列
up

# 查看 CPU 使用率
rate(node_cpu_seconds_total[5m])

# 查看内存使用情况
node_memory_MemTotal_bytes - node_memory_MemFree_bytes - node_memory_Buffers_bytes - node_memory_Cached_bytes

# 查看磁盘使用情况
(node_filesystem_size_bytes - node_filesystem_free_bytes) / node_filesystem_size_bytes * 100

# 查看最近 1 小时的请求速率
rate(http_requests_total[1h])

2. 设置告警规则 ​

创建 alert.rules.yml：

yaml
groups:
- name: example
  rules:
  - alert: InstanceDown
    expr: up == 0
    for: 1m
    labels:
      severity: page
    annotations:
      summary: "Instance {{ $labels.instance }} down"
      description: "{{ $labels.instance }} of job {{ $labels.job }} has been down for more than 1 minute."

3. 备份和恢复数据 ​

bash
# 停止 Prometheus
sudo systemctl stop prometheus

# 备份数据目录
tar czf prometheus-backup-$(date +%Y%m%d).tar.gz /var/lib/prometheus/

# 恢复数据
sudo systemctl stop prometheus
rm -rf /var/lib/prometheus/*
tar xzf prometheus-backup-YYYYMMDD.tar.gz -C /
sudo chown -R prometheus:prometheus /var/lib/prometheus/
sudo systemctl start prometheus

最佳实践 ​

  1. 配置管理：将配置文件纳入版本控制，使用环境变量或配置管理工具管理环境特定的配置
  2. 数据保留：合理设置数据保留时间，平衡存储成本和监控需求
  3. 性能监控：监控 Prometheus 本身的性能指标，确保监控系统健康运行
  4. 高可用性：考虑部署多个 Prometheus 实例，使用联邦或分片方式提高可用性
  5. 安全性：保护 Prometheus 端点，特别是启用 Admin API 的情况下
  6. 告警管理：建立有效的告警管理流程，避免告警疲劳
  7. 文档化：记录监控指标含义和阈值设定原因
