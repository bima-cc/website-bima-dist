常用 Terraform 命令技术文档 ​

简介 ​

Terraform 是由 HashiCorp 公司开发的基础设施即代码（Infrastructure as Code，IaC）工具，允许用户使用声明式的配置文件来定义和管理基础设施。Terraform 支持多云环境（AWS、Azure、Google Cloud、阿里云等），通过统一的语言（HashiCorp Configuration Language，HCL）来描述基础设施状态。

Terraform 的核心概念包括：

  • Provider：提供对底层 API 的访问，如 AWS、Azure、Google Cloud 等
  • Resource：代表基础设施中的组件，如虚拟机、网络、存储等
  • State：记录基础设施的当前状态，用于同步配置与实际状态
  • Module：封装可重用的 Terraform 配置单元

Terraform 广泛应用于多云管理、环境一致性、基础设施版本控制、自动化部署等场景。

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 下载 GPG 密钥
wget -O- https://apt.releases.hashicorp.com/gpg | gpg --dearmor | sudo tee /usr/share/keyrings/hashicorp-archive-keyring.gpg

# 验证密钥指纹
gpg --no-default-keyring --keyring /usr/share/keyrings/hashicorp-archive-keyring.gpg --fingerprint

# 添加仓库
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list

# 更新包索引
sudo apt update

# 安装 Terraform
sudo apt install terraform

# 验证安装
terraform version

初始化命令 ​

bash
# 初始化工作目录，下载配置文件中指定的 provider 插件
terraform init

# 重新初始化并刷新本地插件缓存
terraform init -upgrade

# 从源代码获取模块（如果配置了 backend 存储后）
terraform init -from-module=MODULE-SOURCE

计划与执行命令 ​

bash
# 查看即将执行的操作（不实际执行）
terraform plan

# 保存计划到文件
terraform plan -out=tfplan

# 针对特定资源执行计划
terraform plan -target=RESOURCE_NAME

# 执行配置文件（创建/更新基础设施）
terraform apply

# 应用保存的计划文件
terraform apply tfplan

# 自动确认应用（跳过交互式确认）
terraform apply -auto-approve

# 只销毁资源而不创建新资源
terraform apply -destroy

状态管理命令 ​

bash

terraform show

# 显示状态的详细信息
terraform show -json

# 列出所有资源
terraform state list

# 显示特定资源的状态
terraform state show RESOURCE_NAME

# 移除资源（但不销毁实际资源）
terraform state rm RESOURCE_NAME

# 移动资源状态
terraform state mv OLD_RESOURCE NEW_RESOURCE

# 强制锁定状态（用于解决冲突）
terraform force-unlock LOCK_ID

最佳实践 ​

  1. 版本控制：始终将 .tf 文件纳入版本控制系统，但排除 terraform.tfstate 和 .terraform/ 目录
  2. 远程状态：在团队协作环境中使用远程后端存储状态文件
  3. 变量文件：使用 .tfvars 文件管理敏感信息和环境特定的变量
  4. 模块化：将可重用的配置封装成模块
  5. 定期刷新：定期运行 terraform refresh 以确保状态与实际基础设施同步
  6. 计划预览：始终在应用前运行 terraform plan 以预览变更
  7. 备份状态：定期备份状态文件以防止数据丢失
