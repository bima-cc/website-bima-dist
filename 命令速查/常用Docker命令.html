简介 ​

Docker 是一个开源的应用容器引擎，基于 Go 语言并遵从 Apache2.0 协议开源。Docker 可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。

容器是完全使用沙箱机制，相互之间不会有任何接口。Docker 的核心组件包括：

  • 镜像（Image）：Docker 的只读模板，用于创建容器
  • 容器（Container）：镜像的运行实例，可以被启动、停止、删除等
  • 仓库（Repository）：集中存储镜像的地方，如 Docker Hub

Docker 广泛应用于微服务架构、持续集成/持续部署、开发环境一致性、云原生应用等领域。

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 更新包索引
sudo apt update

# 安装必要的包
sudo apt install apt-transport-https ca-certificates curl gnupg lsb-release

# 添加 Docker 官方 GPG 密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 添加 Docker 仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 更新包索引
sudo apt update

# 安装 Docker Engine
sudo apt install docker-ce docker-ce-cli containerd.io

# 启动 Docker 服务
sudo systemctl start docker

# 设置开机自启
sudo systemctl enable docker

# 将当前用户添加到 docker 组（避免使用 sudo）
sudo usermod -aG docker $USER
# 注销并重新登录以使更改生效

Docker 容器操作 ​

bash
# 运行容器
docker run nginx
docker run -d nginx                    # 后台运行
docker run -it ubuntu:20.04            # 交互式运行
docker run -p 8080:80 nginx            # 端口映射
docker run -v /host/path:/container/path ubuntu    # 挂载卷
docker run --name my-container nginx   # 指定容器名称

# 查看容器
docker ps                              # 查看运行中的容器
docker ps -a                           # 查看所有容器
docker container ls

# 启动、停止、重启容器
docker start container_name
docker stop container_name
docker restart container_name

# 删除容器
docker rm container_name
docker rm -f container_name            # 强制删除运行中的容器
docker container prune                 # 删除所有停止的容器

# 进入正在运行的容器
docker exec -it container_name /bin/bash
docker attach container_name

# 查看容器日志
docker logs container_name
docker logs -f container_name          # 实时查看日志
docker logs --tail 100 container_name  # 查看最后100行日志

# 查看容器资源使用情况
docker stats container_name

# 查看容器详细信息
docker inspect container_name

# 复制文件到/从容器
docker cp /host/file.txt container_name:/container/path/
docker cp container_name:/container/file.txt /host/path/

# 查看容器进程
docker top container_name

Docker Compose ​

Docker Compose 是一个用于定义和运行多容器 Docker 应用程序的工具。通过 Compose，可以使用 YAML 文件来配置应用程序的服务，然后使用一个命令即可从配置中创建并启动所有服务。

Docker Compose 的主要特点包括：

  • 简化多容器应用的部署
  • 通过单一配置文件管理所有服务
  • 支持服务间依赖关系定义
  • 提供环境变量管理
  • 支持卷和网络配置
  • 便于开发、测试和生产环境的一致性

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​
bash
# 下载 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 添加执行权限
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version

高级配置选项 ​

yaml
version: '3.8'

services:
  web:
    # 构建镜像
    build:
      context: .
      dockerfile: Dockerfile.web
      args:
        - BUILD_ARG=value
    
    # 端口映射
    ports:
      - "80:80"
      - "443:443"
    
    # 卷映射
    volumes:
      - ./app:/app
      - static_volume:/static
      - /host/path:/container/path:ro  # 只读
    
    # 环境变量
    environment:
      - ENV_VAR=value
    env_file:
      - .env
    
    # 重启策略
    restart: unless-stopped
    
    # 依赖关系
    depends_on:
      - redis
      - db
    
    # 健康检查
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
    
    # 资源限制
    deploy:
      resources:
        limits:
          cpus: '0.50'
          memory: 512M
        reservations:
          cpus: '0.25'
          memory: 256M
    
    # 网络
    networks:
      - frontend
      - backend

  redis:
    image: redis:alpine
    networks:
      - backend
    command: redis-server --appendonly yes

  db:
    image: postgres:13
    environment:
      POSTGRES_DB: mydb
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
  static_volume:

networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge

最佳实践 ​

  1. 使用 .dockerignore 文件：避免将不必要的文件添加到镜像中
  2. 最小化层数：合并多个命令到一个 RUN 指令中
  3. 使用具体标签：避免使用 latest 标签
  4. 多阶段构建：减少最终镜像大小
  5. 安全扫描：定期扫描镜像漏洞
  6. 健康检查：在 Dockerfile 中添加 HEALTHCHECK 指令
  7. 资源限制：为容器设置适当的资源限制
