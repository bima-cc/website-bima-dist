常用 Git 命令技术文档 ​

简介 ​

Git 是一个免费开源的分布式版本控制系统，由 Linux 内核之父 Linus Torvalds 创建。它被设计用来处理从小到大各种项目的版本管理，具有速度快、支持离线工作、分支操作高效等特点。

Git 的核心概念包括：

  • 工作区（Working Directory）：你正在编辑的文件
  • 暂存区（Staging Area）：准备提交的文件变更
  • 本地仓库（Local Repository）：本地的 Git 仓库
  • 远程仓库（Remote Repository）：远程服务器上的 Git 仓库

Git 广泛应用于软件开发、文档管理和团队协作中，已成为现代开发流程中不可或缺的工具。

语法 ​

Git 命令的基本语法结构：

git <command> [options] [arguments]
1

常用选项：

  • -h, --help: 显示帮助信息
  • -v, --verbose: 详细输出
  • --global: 对当前用户全局设置
  • --local: 对当前仓库设置（默认）

分支合并策略 ​

bash

git checkout feature-branch
git rebase main
git checkout main
git merge feature-branch

git merge -s recursive -X ours feature-branch  # 偏向当前分支

高级历史操作 ​

提交历史管理 ​

bash
# 查看提交历史

git log --graph                   # 图形化显示

# 恢复特定文件到特定版本

git restore --source=commit-hash file # 新语法恢复文件

钩子系统 ​

Git 钩子管理 ​

bash
# Git钩子目录
# .git/hooks/

# 客户端钩子
# pre-commit: 提交前触发
# prepare-commit-msg: 准备提交消息时触发
# commit-msg: 提交消息完成后触发
# post-commit: 提交完成后触发
# pre-rebase: 变基前触发
# post-checkout: 检出后触发
# post-merge: 合并后触发

# 服务端钩子
# pre-receive: 接收推送前触发
# update: 推送过程中每个分支触发
# post-receive: 推送完成后触发

# 创建钩子示例
cat > .git/hooks/pre-commit << 'EOF'
#!/bin/sh
# 检查代码风格
if ! npm run lint; then
    echo "Lint failed, commit aborted"
    exit 1
fi
EOF

chmod +x .git/hooks/pre-commit

配置和别名 ​

bash
# 设置全局配置
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
git config --global core.editor "vim"
git config --global merge.tool "vimdiff"

git config --global alias.tree "log --oneline --graph --all"

git config --global alias.wtf "status"

git config --global credential.helper 'cache --timeout=3600'  # 自定义超时

日常操作技巧 ​

bash
# 查看差异
git diff --cached              # 查看已暂存的更改
git diff HEAD~1               # 查看与上一提交的差异
git diff branch1..branch2     # 比较两个分支

# 临时保存更改
git stash                     # 临时保存未提交的更改
git stash pop                 # 恢复最近一次stash
git stash list                # 查看stash列表
git stash apply stash@{1}     # 应用特定的stash
git stash drop stash@{1}      # 删除特定的stash

# 选择性暂存
git add -p                    # 交互式暂存部分更改
git add --patch               # 同上

# 重写历史
git commit --amend            # 修改最后一次提交
git rebase -i HEAD~3          # 交互式变基，修改最近3次提交

# 子模块操作
git submodule add <url> <path>  # 添加子模块
git submodule update --init --recursive  # 初始化子模块
git submodule foreach git pull origin main  # 更新所有子模块

# 清理操作
git clean -n                  # 预览将要删除的未跟踪文件
git clean -f                  # 删除未跟踪文件
git clean -fd                 # 删除未跟踪文件和目录

高级技巧 ​

bash
# 查找特定更改
git log -S "functionName"     # 查找引入特定字符串的提交
git log -G "regex_pattern"    # 查找涉及正则表达式的提交
git blame filename            # 查看每行的最后修改提交

# 二分查找问题
git bisect start
git bisect bad                # 标记当前版本为坏版本
git bisect good v1.0          # 标记已知好版本
# 测试并标记 good/bad 直到找到问题提交
git bisect reset              # 结束二分查找

# 工作流技巧
git worktree list             # 查看工作树
git worktree add ../feature   # 添加新的工作树
git worktree remove <path>    # 删除工作树

# 远程仓库技巧
git remote set-url origin <new_url>  # 更改远程URL
git remote -v                 # 查看远程仓库信息
git remote prune origin       # 清理过期的远程分支引用

# 性能优化
git repack -d                 # 压缩仓库
git gc --aggressive           # 强制垃圾回收
git prune                     # 清理不可达对象

最佳实践 ​

  1. 使用 .gitignore 文件：避免将不必要的文件添加到仓库中
  2. 频繁提交：保持提交粒度适中，便于回溯和管理
  3. 有意义的提交信息：编写清晰、简洁的提交说明
  4. 分支管理：合理使用分支进行功能开发
  5. 定期同步：经常与远程仓库同步，减少冲突
  6. 备份重要更改：在重大操作前做好备份
  7. 熟悉高级功能：掌握变基、子模块、钩子等高级功能
  8. 选择合适的工作流：根据团队需求选择合适的Git工作流
  9. 保护主分支：设置分支保护规则，确保代码质量
  10. 定期清理：及时清理不需要的分支和大文件
