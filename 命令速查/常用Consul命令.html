常用 Consul 命令技术文档 ​

简介 ​

Consul 是由 HashiCorp 公司开发的分布式服务网格解决方案，提供服务发现、配置管理和健康检查等功能。Consul 采用 Raft 一致性算法，支持多数据中心部署，具有高可用性和强一致性特点。Consul 可以在物理机、虚拟机和容器环境中运行，是现代微服务架构的重要组件。

Consul 的核心功能包括：

  • 服务发现：客户端可以注册服务并通过 DNS 或 HTTP 接口发现其他服务
  • 健康检查：支持对节点和服务进行健康检查，确保流量只路由到健康的节点
  • KV 存储：提供键值对存储，用于动态配置、特性标记等
  • 多数据中心：内置对多数据中心的支持，无需复杂配置
  • 安全服务通信：通过 Connect 功能实现服务间的安全通信

Consul 广泛应用于微服务架构、服务网格、配置管理、服务发现等场景。

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 下载 GPG 密钥
wget -O- https://apt.releases.hashicorp.com/gpg | gpg --dearmor | sudo tee /usr/share/keyrings/hashicorp-archive-keyring.gpg

# 验证密钥指纹
gpg --no-default-keyring --keyring /usr/share/keyrings/hashicorp-archive-keyring.gpg --fingerprint

# 添加仓库
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list

# 更新包索引
sudo apt update

# 安装 Consul
sudo apt install consul

# 验证安装

Docker 方式安装 ​

bash
# 运行单节点 Consul
docker run -d --name=dev-consul -p 8500:8500 -p 8600:8600/udp consul agent -dev -client=0.0.0.0

# 运行 Consul UI
docker run -d --name=consul-ui -p 8500:8500 consul agent -dev -ui -client=0.0.0.0

# 使用 docker-compose 运行 Consul 集群
cat <<EOF > docker-compose.yml
version: '3'
services:
  consul-server1:
    image: consul:latest
    restart: always
    networks:
      - consul
    ports:
      - "8300:8300"
      - "8301:8301"
      - "8301:8301/udp"
      - "8500:8500"
      - "8600:8600/udp"
    command: "agent -server -bootstrap-expect=1 -data-dir=/tmp/consul -bind=0.0.0.0 -client=0.0.0.0 -ui"

networks:
  consul:
    driver: bridge
EOF

# 启动服务
docker-compose up -d

server = true

# 数据目录
data_dir = "/opt/consul/data"

  dns = 8600

Consul Agent 启动命令 ​

启动服务器模式 ​

bash
# 启动开发模式（单节点，非持久化数据）

# 启动服务器模式
consul agent -server -bootstrap-expect=1 -data-dir=/tmp/consul -bind=127.0.0.1 -node=server-1 -enable-script-checks=true

# 使用配置文件启动服务器
consul agent -config-dir=/etc/consul.d -data-dir=/tmp/consul -server -bind=127.0.0.1

注册服务 ​

bash
# 通过 HTTP API 注册服务

    "tags": ["rails"],
    "port": 80,

      "id": "web-check-1",
      "name": "Web Health Check",
      "http": "http://localhost:80/status",
      "interval": "10s",
      "timeout": "1s"
    }
  }
}
EOF

# 将配置文件放在自动加载目录
sudo cp web.json /etc/consul.d/

操作 KV 存储 ​

bash

consul kv list app/web/

# 删除键

ACL 策略管理 ​

bash
# 创建 ACL 策略
cat <<EOF > global-management-policy.hcl
name = "global-management"
rules = """
operator = "write"
agent = "write"
node = "write"
service = "write"
session = "write"
query = "write"
acl = "write"
sentinel = "write"
keyring = "write"
telemetry = "write"
event = "write"
molecule = "write"
coordinate = "write"
peer = "write"
partition = "write"
namespace = "write"
"""
EOF

# 创建策略
consul acl policy write -name="global-management" -rules=@global-management-policy.hcl

# 创建令牌
consul acl token create -policy-name="global-management"

最佳实践 ​

  1. 部署拓扑：建议使用奇数个服务器节点（通常是 3 或 5 个）以确保高可用性
  2. ACL 策略：在生产环境中启用 ACL 系统并实施最小权限原则
  3. 备份策略：定期备份 Consul 数据目录和快照
  4. 监控告警：监控集群健康状态、Raft 指标和业务服务健康状况
  5. 安全通信：配置加密通信以保护数据传输安全
  6. 配置管理：使用配置文件而非命令行参数管理复杂配置
  7. 版本管理：及时升级 Consul 版本以获得新功能和安全修复
