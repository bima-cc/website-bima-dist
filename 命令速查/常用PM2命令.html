常用 PM2 命令技术文档 ​

简介 ​

PM2 (Process Manager 2) 是一个功能强大的 Node.js 应用程序进程管理器，具有内置的负载均衡器。它可以保持应用程序永久运行，支持零停机重载，并提供简单的命令来管理进程。PM2 能够在应用程序崩溃时自动重启，提供高级部署选项，并且能够生成内置的监控仪表板。

PM2 的主要特点包括：

  • 守护进程管理：自动重启崩溃的应用程序
  • 负载均衡：内置负载均衡器，支持多核 CPU
  • 零停机重载：无需停机即可更新应用程序
  • 进程监控：实时监控 CPU 和内存使用情况
  • 配置文件支持：使用 JSON 配置文件管理复杂部署
  • 日志管理：集中管理和查看应用程序日志
  • 部署管理：支持一键部署到不同环境

PM2 广泛应用于生产环境中部署和管理 Node.js 应用程序，特别适用于需要高可用性和持续运行的 Web 服务。

语法 ​

PM2 命令的基本语法结构：

pm2 [command] [options] [app_name|app_id|all]
1

常用选项：

  • -f, --force: 强制执行操作
  • -m, --merge-args: 合并配置文件中的 args
  • -i, --instances <number>: 启动指定数量的实例
  • -n, --name <name>: 为应用指定名称
  • -w, --watch: 监听文件变化并自动重启
  • -e, --error <path>: 错误日志文件路径
  • -o, --output <path>: 输出日志文件路径
  • -l, --log <path>: 所有日志文件路径
  • --env <environment>: 指定环境配置

常用命令 ​

应用程序启动和管理 ​

bash
# 启动应用程序
pm2 start app.js
pm2 start app.js --name "my-app"

# 启动多个实例（集群模式）
pm2 start app.js -i 4  # 启动 4 个实例
pm2 start app.js -i max  # 启动与 CPU 核心数相同的实例

# 启动应用程序并监听文件变化
pm2 start app.js --watch

# 启动应用程序并传入参数
pm2 start app.js -- arg1 arg2
pm2 start app.js --name "api-server" -- -p 8080 -e production

# 使用配置文件启动
pm2 start ecosystem.config.js
pm2 start ./path/to/ecosystem.config.js

# 启动 JSON 配置文件中的所有应用
pm2 start ecosystem.json

应用程序控制 ​

bash
# 停止应用程序
pm2 stop app_name
pm2 stop app_id
pm2 stop all

# 重启应用程序
pm2 restart app_name
pm2 restart app_id
pm2 restart all

# 重新加载应用程序（零停机时间重启）
pm2 reload app_name
pm2 reload all

# 删除应用程序
pm2 delete app_name
pm2 delete app_id
pm2 delete all

# 保存当前进程列表
pm2 save

# 恢复之前保存的进程列表
pm2 resurrect

# 重新启动 PM2 守护进程
pm2 update

常用配置文件示例 ​

Ecosystem 配置文件 (ecosystem.config.js) ​

javascript
module.exports = {
  apps: [{
    name: 'API-Server',
    script: './app.js',
    instances: 'max',
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'development',
      PORT: 3000
    },
    env_production: {
      NODE_ENV: 'production',
      PORT: 8080
    },
    error_file: './logs/err.log',
    out_file: './logs/out.log',
    log_file: './logs/combined.log',
    time: true,
    watch: true,
    ignore_watch: ['node_modules', 'logs'],
    max_memory_restart: '1G',
    node_args: '--harmony',
    args: ['--port', '8080']
  }, {
    name: 'Worker',
    script: './worker.js',
    instances: 2,
    exec_mode: 'fork',
    env: {
      NODE_ENV: 'development',
      WORKER_TYPE: 'processor'
    }
  }]
};

最佳实践 ​

  1. 使用配置文件：使用 ecosystem.config.js 管理复杂部署配置
  2. 集群模式：在生产环境中使用集群模式充分利用多核 CPU
  3. 内存限制：设置内存限制以防止内存泄漏导致系统崩溃
  4. 日志管理：配置日志轮转以避免磁盘空间耗尽
  5. 环境变量：使用不同环境的配置文件管理环境变量
  6. 开机自启：配置 PM2 开机自启以确保应用持续运行
  7. 监控告警：定期检查应用状态和资源使用情况
  8. 版本控制：将配置文件纳入版本控制
  9. 安全配置：不在配置文件中存储敏感信息
  10. 定期更新：定期更新 PM2 以获得新功能和安全补丁
