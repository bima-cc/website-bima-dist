常用 Redis 命令技术文档 ​

简介 ​

Redis（Remote Dictionary Server）是一个开源的内存数据结构存储系统，可以用作数据库、缓存和消息中间件。Redis 支持多种数据结构，如字符串（strings）、哈希（hashes）、列表（lists）、集合（sets）、有序集合（sorted sets）等，具有高性能、丰富的功能和简单的 API。

Redis 的主要特点包括：

  • 基于内存的数据存储，提供极高的性能
  • 支持多种数据结构
  • 支持数据持久化
  • 支持主从复制
  • 支持事务和 Lua 脚本
  • 丰富的客户端库支持

Redis 广泛应用于缓存系统、会话存储、排行榜、消息队列、实时分析等场景。

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 更新包索引
sudo apt update

# 安装 Redis 服务器
sudo apt install redis-server

# 启动 Redis 服务
sudo systemctl start redis-server

# 设置开机自启
sudo systemctl enable redis-server

# 检查 Redis 服务状态
sudo systemctl status redis-server

# 验证安装
redis-server --version
redis-cli --version

语法 ​

Redis 命令的基本语法结构：

COMMAND [ARGUMENTS]
1

Redis CLI 的基本语法结构：

redis-cli [OPTIONS] [COMMAND [ARGUMENTS]]
1

常用选项：

  • -h HOST: 指定服务器主机名
  • -p PORT: 指定服务器端口
  • -a PASSWORD: 指定密码
  • -n DATABASE: 指定数据库编号
  • -c: 集群模式
  • --raw: 输出原始格式
  • -v, --version: 显示版本信息

最佳实践 ​

  1. 内存管理：合理设置最大内存限制和淘汰策略
  2. 持久化策略：根据业务需求选择合适的持久化方式（RDB/AOF）
  3. 安全配置：设置密码认证，限制网络访问
  4. 键命名规范：使用有意义的键名，建议使用冒号分隔层级
  5. 过期时间：为缓存数据设置合适的过期时间
  6. 批量操作：使用批量命令提高性能（MGET/MSET）
  7. 连接池：在应用中使用连接池管理 Redis 连接
  8. 监控告警：监控内存使用率、命中率等关键指标
  9. 备份策略：定期备份 Redis 数据
  10. 性能调优：根据业务特点调整 Redis 配置参数
