常用 Kafka 命令技术文档 ​

简介 ​

Apache Kafka 是一个分布式流处理平台，由 LinkedIn 开发并于 2011 年开源，后来成为 Apache 软件基金会的顶级项目。Kafka 以其高性能、高吞吐量、低延迟和可扩展性著称，广泛用于构建实时数据管道和流应用。

Kafka 的核心概念包括：

  • Producer：发布消息到 Kafka 集群的客户端应用
  • Consumer：订阅并消费 Kafka 集群中消息的客户端应用
  • Broker：Kafka 集群中的服务器节点
  • Topic：消息的分类标识，类似邮箱名称
  • Partition：Topic 的分区，提高并行处理能力
  • Offset：消息在分区中的唯一标识符
  • Zookeeper：存储 Kafka 集群元数据（Kafka 3.0+ 版本开始逐步替代）

Kafka 广泛应用于日志收集、流处理、事件溯源、消息系统、数据同步等场景。

Docker 方式安装 ​

bash
# 运行单节点 Kafka
docker run --name kafka -d -p 9092:9092 \
  -e KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://localhost:9092 \
  -e KAFKA_ZOOKEEPER_CONNECT=zookeeper:2181 \
  -e KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR=1 \
  wurstmeister/kafka

# 使用 docker-compose 运行完整的 Kafka 环境
cat <<EOF > docker-compose.yml
version: '3'
services:
  zookeeper:
    image: confluentinc/cp-zookeeper:latest
    environment:
      ZOOKEEPER_CLIENT_PORT: 2181
      ZOOKEEPER_TICK_TIME: 2000

  kafka:
    image: confluentinc/cp-kafka:latest
    depends_on:
      - zookeeper
    ports:
      - "9092:9092"
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
EOF

# 启动服务
docker-compose up -d

基本配置 ​

服务器配置文件 (server.properties) ​

properties
# broker.id 必须唯一
broker.id=0

# 监听地址
listeners=PLAINTEXT://:9092

# Advertised listeners
advertised.listeners=PLAINTEXT://localhost:9092

# 日志目录
log.dirs=/tmp/kafka-logs

# Zookeeper 连接
zookeeper.connect=localhost:2181

# 分区数默认值
num.partitions=1

# 默认副本因子
default.replication.factor=1

# 最小同步副本数
min.insync.replicas=1

重置消费者组偏移量 ​

bash
# 重置到最早偏移量
bin/kafka-consumer-groups.sh --bootstrap-server localhost:9092 \
  --group my-group \
  --reset-offsets \
  --to-earliest \
  --topic my-topic \
  --execute

# 重置到最晚偏移量
bin/kafka-consumer-groups.sh --bootstrap-server localhost:9092 \
  --group my-group \
  --reset-offsets \
  --to-latest \
  --topic my-topic \
  --execute

# 重置到指定时间戳
bin/kafka-consumer-groups.sh --bootstrap-server localhost:9092 \
  --group my-group \
  --reset-offsets \
  --to-datetime 2023-01-01T00:00:00 \
  --topic my-topic \
  --execute

# 重置到指定偏移量
bin/kafka-consumer-groups.sh --bootstrap-server localhost:9092 \
  --group my-group \
  --reset-offsets \
  --to-offset 1000 \
  --topic my-topic \
  --execute

最佳实践 ​

  1. Topic 设计：合理设计 Topic 名称和分区数，考虑数据分布和并行处理需求
  2. 数据保留策略：根据业务需求设置合适的数据保留时间和大小限制
  3. 监控告警：监控消费者滞后、Broker 健康状态、磁盘使用情况等关键指标
  4. 安全配置：在生产环境中配置 SSL/TLS 加密和认证授权
  5. 容量规划：根据预期消息量和吞吐量规划集群规模
  6. 备份策略：制定适当的数据备份和恢复策略
  7. 版本管理：及时升级 Kafka 版本以获得新功能和安全修复
