常用 Kubernetes 命令技术文档 ​

简介 ​

Kubernetes（简称 K8s）是一个开源的容器编排平台，用于自动化部署、扩展和管理容器化应用程序。Kubernetes 由 Google 设计，并捐赠给 Cloud Native Computing Foundation (CNCF)，现已成为容器编排领域的事实标准。

Kubernetes 的核心功能包括：

  • 自动化容器部署和复制
  • 水平扩展或收缩容器规模
  • 命名和发现服务
  • 负载均衡
  • 滚动更新和回滚
  • 存储编排
  • 自我修复机制

Kubernetes 广泛应用于微服务架构、云原生应用、DevOps 流程和大规模容器化应用部署中。

安装方法 ​

安装 kubectl（Kubernetes 命令行工具） ​

Linux 系统 ​

bash
# 使用 curl 安装
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

# 验证二进制文件（可选）
curl -LO "https://dl.k8s.io/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl.sha256"
echo "$(cat kubectl.sha256)  kubectl" | sha256sum --check

# 使 kubectl 二进制文件可执行
chmod +x kubectl

# 移动到 PATH 中的目录
sudo mv kubectl /usr/local/bin/

# 验证安装
kubectl version --client

语法 ​

Kubernetes 命令的基本语法结构：

kubectl [command] [TYPE] [NAME] [flags]
1

常用通用选项：

  • -h, --help: 显示帮助信息
  • -v, --v=LEVEL: 设置日志详细程度
  • -n, --namespace=NAMESPACE: 指定命名空间
  • --context=CONTEXT: 指定上下文
  • -o, --output=OUTPUT_FORMAT: 指定输出格式

Pod 操作 ​

bash
# 运行单个 Pod
kubectl run POD_NAME --image=IMAGE_NAME

# 查看 Pod 日志
kubectl logs POD_NAME
kubectl logs -f POD_NAME            # 实时查看日志
kubectl logs -c CONTAINER_NAME POD_NAME  # 查看特定容器日志
kubectl logs --previous POD_NAME    # 查看之前容器的日志

# 进入 Pod
kubectl exec -it POD_NAME -- /bin/sh
kubectl exec -it POD_NAME -c CONTAINER_NAME -- /bin/bash

# 复制文件
kubectl cp POD_NAME:/path/to/file /local/path  # 从 Pod 复制到本地
kubectl cp /local/path POD_NAME:/path/to/file  # 从本地复制到 Pod

插件管理 ​

bash
# 查看插件
kubectl plugin list

# 安装 krew（Kubernetes 包管理器）
(
  set -x; cd "$(mktemp -d)" &&
  OS="$(uname | tr '[:upper:]' '[:lower:]')" &&
  ARCH="$(uname -m | sed -e 's/x86_64/amd64/' -e 's/\(arm\)\(64\)\?.*/\1\2/' -e 's/aarch64$/arm64/')" &&
  curl -fsSLO "https://github.com/kubernetes-sigs/krew/releases/latest/download/krew-$OS_$ARCH.tar.gz" &&
  tar zxvf krew-$OS_$ARCH.tar.gz &&
  KREW=./krew-$OS_$ARCH ./krew-$OS_$ARCH install krew
)

# 使用 krew 安装插件
kubectl krew install view-secret
kubectl krew install whoami

最佳实践 ​

  1. 使用配置文件：通过 YAML/JSON 文件管理资源，而不是命令行
  2. 命名规范：使用有意义的名称和标签组织资源
  3. 命名空间隔离：使用命名空间隔离不同环境或团队
  4. 资源限制：为 Pod 设置适当的资源请求和限制
  5. 健康检查：配置存活探针和就绪探针
  6. 安全配置：使用 RBAC 控制访问权限，避免使用默认服务账户
  7. 配置分离：将配置与镜像分离，使用 ConfigMap 和 Secret
  8. 版本控制：将 Kubernetes 配置文件纳入版本控制
  9. 监控日志：配置适当的监控和日志记录
  10. 备份策略：制定 etcd 和持久化数据的备份策略
