常用 MySQL 命令技术文档 ​

简介 ​

MySQL 是世界上最受欢迎的开源关系型数据库管理系统之一，由 Oracle 公司开发和维护。MySQL 使用结构化查询语言（SQL）进行数据库操作，以其高性能、高可靠性和易于使用的特点被广泛应用于各种规模的应用程序中。

MySQL 的主要特点包括：

  • 高性能和高可用性
  • 支持多种存储引擎
  • 丰富的数据类型
  • 强大的事务支持
  • 良好的跨平台兼容性
  • 丰富的社区支持

MySQL 广泛应用于 Web 应用程序、数据仓库、日志系统、在线交易处理等多种场景。

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 更新包索引
sudo apt update

# 安装 MySQL 服务器
sudo apt install mysql-server

# 安装 MySQL 客户端（通常随服务器一起安装）
sudo apt install mysql-client

# 启动 MySQL 服务
sudo systemctl start mysql

# 设置开机自启
sudo systemctl enable mysql

# 运行安全配置脚本
sudo mysql_secure_installation

# 验证安装
mysql --version
sudo systemctl status mysql

语法 ​

MySQL 命令的基本语法结构：

MySQL 客户端连接 ​

mysql [options] [database_name]
1

SQL 语句语法 ​

COMMAND [CLAUSES] [CONDITIONS];
1

常用选项：

  • -h, --host=name: 指定主机名
  • -u, --user=name: 指定用户名
  • -p, --password[=name]: 指定密码
  • -P, --port=#: 指定端口号
  • -D, --database=name: 指定数据库
  • -e, --execute=name: 执行 SQL 语句

表操作 ​

sql
-- 显示当前数据库中的所有表
SHOW TABLES;

-- 显示表结构
DESCRIBE table_name;
DESC table_name;
SHOW COLUMNS FROM table_name;

-- 显示表创建语句

-- 创建表
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE,
    age INT DEFAULT 0,

ALTER TABLE table_name DROP COLUMN column_name;
ALTER TABLE table_name MODIFY COLUMN column_name new_datatype;
ALTER TABLE table_name CHANGE COLUMN old_name new_name datatype;

-- 重命名表
RENAME TABLE old_name TO new_name;

-- 查看表状态
SHOW TABLE STATUS;

sql

用户管理 ​

sql
-- 显示所有用户
SELECT User, Host FROM mysql.user;

-- 创建用户
CREATE USER 'username'@'localhost' IDENTIFIED BY 'password';
CREATE USER 'username'@'%' IDENTIFIED BY 'password';  -- 允许远程连接

-- 删除用户
DROP USER 'username'@'localhost';

-- 修改用户密码
ALTER USER 'username'@'localhost' IDENTIFIED BY 'new_password';
SET PASSWORD FOR 'username'@'localhost' = PASSWORD('new_password');  -- 旧版本

-- 授权
GRANT SELECT, INSERT ON database_name.table_name TO 'username'@'localhost';
GRANT ALL PRIVILEGES ON database_name.* TO 'username'@'localhost';
GRANT ALL PRIVILEGES ON *.* TO 'username'@'localhost';  -- 所有数据库

-- 撤销权限
REVOKE SELECT ON database_name.table_name FROM 'username'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;

备份和恢复 ​

bash
# 备份整个数据库
mysqldump -u username -p database_name > backup.sql

# 备份多个数据库
mysqldump -u username -p --databases db1 db2 db3 > backup.sql

# 备份所有数据库
mysqldump -u username -p --all-databases > all_backup.sql

# 备份特定表
mysqldump -u username -p database_name table1 table2 > tables_backup.sql

# 备份结构（不包含数据）
mysqldump -u username -p --no-data database_name > structure.sql

# 备份数据（不包含结构）
mysqldump -u username -p --no-create-info database_name > data_only.sql

# 恢复数据库
mysql -u username -p database_name < backup.sql

# 使用 source 命令恢复
mysql> USE database_name;
mysql> SOURCE /path/to/backup.sql;

最佳实践 ​

  1. 使用合适的数据类型：根据实际需求选择最适合的数据类型
  2. 建立适当的索引：为经常查询的字段建立索引，但不要过度索引
  3. 使用事务保护数据完整性：在需要保证数据一致性的操作中使用事务
  4. 定期备份：制定合理的备份策略并定期验证备份的有效性
  5. 安全配置：移除测试用户，设置强密码，限制远程访问
  6. 性能监控：监控查询性能，识别和优化慢查询
  7. 字符集设置：使用 UTF8MB4 字符集以支持完整的 Unicode
  8. 连接池管理：合理设置最大连接数，避免连接耗尽
  9. SQL 注入防护：使用预处理语句或转义特殊字符
  10. 定期维护：执行 ANALYZE TABLE 和 OPTIMIZE TABLE 维护操作
