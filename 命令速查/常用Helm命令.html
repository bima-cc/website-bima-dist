常用 Helm 命令技术文档 ​

简介 ​

Helm 是 Kubernetes 的包管理器，被称为 "Kubernetes 的包管理器"。它允许开发者和运维人员轻松地定义、安装和升级复杂的 Kubernetes 应用程序。Helm 使用称为 Charts 的打包格式，Charts 是描述一组相关 Kubernetes 资源的文件集合。

Helm 的核心概念包括：

  • Chart：一个 Helm 包，包含在 Kubernetes 集群中运行应用程序所需的全部资源定义
  • Repository：存储和分发 Charts 的位置
  • Release：一个 Chart 在 Kubernetes 集群中的具体运行实例
  • Tiller：Helm 2.x 中的服务端组件（Helm 3.x 已移除）

Helm 广泛应用于 Kubernetes 应用部署、应用版本管理、多环境配置管理等场景。

使用包管理器安装（Ubuntu/Debian） ​

bash
# 添加 Helm 仓库的 GPG 密钥
curl https://baltocdn.com/helm/signing.asc | sudo apt-key add -

# 添加 Helm 仓库
sudo apt-get install apt-transport-https --yes
echo "deb https://baltocdn.com/helm/stable/debian/ all main" | sudo tee /etc/apt/sources.list.d/helm-stable-debian.list

# 更新包索引
sudo apt-get update

# 安装 Helm
sudo apt-get install helm

# 验证安装
helm version

Chart 管理命令 ​

bash
# 搜索可用的 Charts
helm search hub KEYWORD          # 搜索 Artifact Hub
helm search repo KEYWORD         # 搜索已添加的仓库

# 添加 Chart 仓库
helm repo add REPO_NAME URL
# 示例
helm repo add bitnami https://charts.bitnami.com/bitnami

# 列出已添加的仓库
helm repo list

# 从仓库更新本地 Chart 信息
helm repo update

# 显示 Chart 详情
helm show chart CHART_NAME
helm show values CHART_NAME
helm show readme CHART_NAME
helm show all CHART_NAME

# 拉取 Chart（不安装）
helm pull CHART_NAME

# 拉取并解压 Chart
helm pull CHART_NAME --untar

# 保存 Chart 到本地目录
helm pull CHART_NAME --destination DIR_PATH --version VERSION

Release 管理命令 ​

bash
# 列出所有 Release
helm list

# 列出指定命名空间的 Release
helm list -n NAMESPACE

# 查看 Release 状态
helm status RELEASE_NAME

# 查看 Release 的详细信息
helm get all RELEASE_NAME

# 查看 Release 的值
helm get values RELEASE_NAME

# 查看 Release 的清单（Kubernetes 清单文件）
helm get manifest RELEASE_NAME

# 查看 Release 的钩子
helm get hooks RELEASE_NAME

# 升级 Release
helm upgrade RELEASE_NAME CHART_PATH

# 升级并设置值
helm upgrade RELEASE_NAME CHART_PATH --set key=value

# 升级并使用值文件
helm upgrade RELEASE_NAME CHART_PATH -f values.yaml

# 回滚到之前的版本
helm rollback RELEASE_NAME REVISION

# 卸载 Release
helm uninstall RELEASE_NAME

# 卸载 Release（旧版本命令）
helm delete RELEASE_NAME

# 查看历史版本
helm history RELEASE_NAME

最佳实践 ​

  1. 版本控制：将 Chart 文件和值文件纳入版本控制系统
  2. 命名空间隔离：为不同的环境或应用使用不同的命名空间
  3. 值文件管理：使用不同的值文件管理不同环境的配置
  4. 定期更新：定期更新仓库和 Charts 以获取最新功能和安全修复
  5. 回滚策略：在升级失败时能够快速回滚到之前的工作版本
  6. 权限管理：合理配置 RBAC 权限，确保安全访问
  7. 监控和日志：监控 Helm 操作和应用运行状态
