常用 Vue 命令技术文档 ​

简介 ​

Vue.js 是一个渐进式 JavaScript 框架，用于构建用户界面。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动。

Vue.js 的主要特点包括：

  • 渐进式框架：可以根据项目需求逐步采用 Vue 的功能
  • 组件化：可复用的组件系统，便于构建复杂界面
  • 响应式数据绑定：数据变化自动更新视图
  • 虚拟 DOM：高效的 DOM 操作和更新机制
  • 轻量级：压缩后体积小，加载速度快
  • 直观的 API：易于学习和使用
  • 强大的生态系统：包含路由、状态管理等官方维护的工具

Vue.js 广泛应用于现代 Web 应用开发，从简单的页面增强到复杂的单页应用（SPA），都是 Vue.js 的典型应用场景。配合 Vue Router 和 Vuex/Pinia，可以构建功能完善的现代化前端应用。

语法 ​

Vue CLI 命令的基本语法结构：

vue [command] [options] [arguments]
1

Vite 项目命令的基本语法结构：

npm run [script-name] [options]
1

常用选项：

  • --version, -V: 显示版本信息
  • --help, -h: 显示帮助信息
  • --mode <mode>: 指定环境模式
  • --host <host>: 指定主机地址
  • --port <port>: 指定端口号
  • --open: 自动打开浏览器
  • --force: 强制依赖预构建

vite.config.js 示例 ​

javascript
import { fileURLToPath, URL } from 'node:url'

import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [
    vue(),
  ],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  },
  server: {
    port: 3000,
    open: true,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        secure: false,
      }
    }
  },
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['vue', 'vue-router'],
        }
      }
    }
  }
})

Vue 3 Composition API 示例 ​

vue
<template>
  <div class="hello">
    <h1>{{ title }}</h1>
    <p>{{ message }}</p>
    <button @click="increment">Count: {{ count }}</button>
  </div>
</template>

<script setup>
import { ref, computed, onMounted } from 'vue'

// 响应式数据
const title = ref('Hello Vue 3!')
const count = ref(0)
const message = computed(() => `Current count is ${count.value}`)

// 方法
const increment = () => {
  count.value++
}

// 生命周期钩子
onMounted(() => {
  console.log('Component mounted!')
})
</script>

<style scoped>
.hello {
  text-align: center;
  padding: 20px;
}
</style>

实用脚本示例 ​

bash
#!/bin/bash
# Vue 项目构建脚本

# 检查是否为 Vue 项目
if [ ! -f "package.json" ]; then
    echo "当前目录不是 Vue 项目"
    exit 1
fi

# 检查依赖是否安装
if [ ! -d "node_modules" ]; then
    echo "安装依赖..."
    npm install
fi

# 运行代码检查
echo "运行代码检查..."
npm run lint

if [ $? -ne 0 ]; then
    echo "代码检查失败，请修复问题后重试"
    exit 1
fi

# 构建项目
echo "构建项目..."
npm run build

if [ $? -eq 0 ]; then
    echo "构建成功！"
    echo "构建产物位于 dist/ 目录"
else
    echo "构建失败！"
    exit 1
fi

最佳实践 ​

  1. 使用 Vite：对于新项目，优先使用 Vite 而不是 Vue CLI，因为构建速度更快
  2. 组件设计：遵循单一职责原则，保持组件简洁和可复用
  3. 状态管理：小型项目使用 props 和 emit，大型项目使用 Pinia 或 Vuex
  4. 路由设计：合理组织路由结构，使用懒加载提升性能
  5. 代码分割：使用动态导入实现代码分割
  6. TypeScript：在大型项目中使用 TypeScript 提升代码质量
  7. 测试驱动：编写单元测试和集成测试
  8. 性能监控：使用 Vue DevTools 监控应用性能
  9. 安全性：防范 XSS 攻击，验证用户输入
  10. 文档化：为组件和功能编写清晰的文档
