常用 Elasticsearch 命令技术文档 ​

简介 ​

Elasticsearch 是一个分布式的 RESTful 搜索和分析引擎，基于 Apache Lucene 构建。它能够快速地存储、搜索和分析大量数据，被广泛应用于全文搜索、结构化搜索、日志分析、安全分析、指标监控等场景。

Elasticsearch 的主要特点包括：

  • 分布式搜索和分析引擎
  • RESTful API 接口
  • 实时搜索能力
  • 水平扩展性
  • 多租户支持
  • 强大的聚合分析功能
  • 与 Logstash 和 Kibana 集成形成 ELK 技术栈

Elasticsearch 广泛应用于企业级搜索、日志分析、安全情报、商业智能、网站搜索和应用性能监控等领域。

安装方法 ​

Linux 系统 ​

Ubuntu/Debian 系统 ​

bash
# 安装 Java（Elasticsearch 依赖 Java 运行时）
sudo apt update
sudo apt install openjdk-11-jdk

# 导入 Elasticsearch PGP 密钥
wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | sudo apt-key add -

# 添加 Elasticsearch 仓库
echo "deb https://artifacts.elastic.co/packages/8.x/apt stable main" | sudo tee -a /etc/apt/sources.list.d/elastic-8.x.list

# 更新包索引
sudo apt update

# 安装 Elasticsearch
sudo apt install elasticsearch

# 启动 Elasticsearch 服务
sudo systemctl start elasticsearch

# 设置开机自启
sudo systemctl enable elasticsearch

# 检查 Elasticsearch 服务状态
sudo systemctl status elasticsearch

# 验证安装
curl -X GET "localhost:9200/"

CentOS/RHEL/Fedora 系统 ​

bash
# 安装 Java
sudo yum install java-11-openjdk-devel    # CentOS/RHEL 7
sudo dnf install java-11-openjdk-devel    # Fedora/CentOS Stream

# 导入 Elasticsearch PGP 密钥
sudo rpm --import https://artifacts.elastic.co/GPG-KEY-elasticsearch

# 添加 Elasticsearch 仓库
sudo tee -a /etc/yum.repos.d/elastic.repo << EOF
[elastic-8.x]
name=Elastic repository for 8.x packages
baseurl=https://artifacts.elastic.co/packages/8.x/yum
gpgcheck=1
gpgkey=https://artifacts.elastic.co/GPG-KEY-elasticsearch
enabled=1
autorefresh=1
type=rpm-md
EOF

# 安装 Elasticsearch
sudo yum install elasticsearch    # CentOS/RHEL 7
sudo dnf install elasticsearch    # Fedora/CentOS Stream

# 启动 Elasticsearch 服务
sudo systemctl start elasticsearch

# 设置开机自启
sudo systemctl enable elasticsearch

# 检查 Elasticsearch 服务状态
sudo systemctl status elasticsearch

# 验证安装
curl -X GET "localhost:9200/"

语法 ​

Elasticsearch 使用 RESTful API，命令的基本语法结构：

HTTP_METHOD http://HOST:PORT/ENDPOINT [REQUEST_BODY]
1

常用 HTTP 方法：

  • GET: 获取资源
  • POST: 创建资源或执行操作
  • PUT: 创建或更新资源
  • DELETE: 删除资源
  • HEAD: 获取资源元信息

搜索操作 ​

bash
# 简单搜索
curl -X GET "localhost:9200/my_index/_search?q=title:elasticsearch&pretty"

# 复杂搜索查询
curl -X GET "localhost:9200/my_index/_search?pretty" -H 'Content-Type: application/json' -d'
{
  "query": {
    "match": {
      "content": "document"
    }
  },
  "highlight": {
    "fields": {
      "content": {}
    }
  }
}'

# 全文搜索
curl -X GET "localhost:9200/my_index/_search?pretty" -H 'Content-Type: application/json' -d'
{
  "query": {
    "multi_match": {
      "query": "elasticsearch",
      "fields": ["title", "content"]
    }
  }
}'

# 精确匹配搜索
curl -X GET "localhost:9200/my_index/_search?pretty" -H 'Content-Type: application/json' -d'
{
  "query": {
    "term": {
      "author.keyword": "John Doe"
    }
  }
}'

# 范围搜索
curl -X GET "localhost:9200/my_index/_search?pretty" -H 'Content-Type: application/json' -d'
{
  "query": {
    "range": {
      "date": {
        "gte": "2023-01-01",
        "lte": "2023-12-31"
      }
    }
  }
}'

# 组合查询
curl -X GET "localhost:9200/my_index/_search?pretty" -H 'Content-Type: application/json' -d'
{
  "query": {
    "bool": {
      "must": [
        { "match": { "content": "elasticsearch" } }
      ],
      "filter": [
        { "term": { "author.keyword": "John Doe" } },
        { "range": { "date": { "gte": "2023-01-01" } } }
      ]
    }
  }
}'

# 排序
curl -X GET "localhost:9200/my_index/_search?pretty" -H 'Content-Type: application/json' -d'
{
  "query": {
    "match_all": {}
  },
  "sort": [
    {
      "date": {
        "order": "desc"
      }
    }
  ]
}'

# 分页
curl -X GET "localhost:9200/my_index/_search?pretty" -H 'Content-Type: application/json' -d'
{
  "query": {
    "match_all": {}
  },
  "from": 0,
  "size": 10
}'

最佳实践 ​

  1. 合理规划索引结构：根据查询模式设计索引和映射
  2. 配置适当的分片数量：避免过多或过少的分片
  3. 使用合适的分析器：根据文本内容选择适当的分析器
  4. 监控集群健康状态：定期检查集群健康和性能指标
  5. 实施安全措施：启用认证和授权，配置 HTTPS
  6. 设置合理的副本数量：平衡性能和可靠性
  7. 定期备份数据：配置快照策略保护数据
  8. 优化查询性能：使用过滤器、聚合和缓存
