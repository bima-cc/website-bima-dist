常用 Podman 命令技术文档 ​

简介 ​

Podman（Pod Manager）是 Red Hat 开发的无守护进程的容器管理工具，作为 Docker 的替代品。Podman 提供了与 Docker 兼容的命令行接口，但不需要后台守护进程，因此更加安全和轻量级。Podman 支持 OCI（Open Container Initiative）兼容的容器和镜像，可以直接运行容器而无需运行守护进程。

Podman 的核心特性包括：

  • 无守护进程：无需后台服务，提高安全性
  • Rootless 模式：支持非 root 用户运行容器
  • Pod 支持：原生支持 Kubernetes Pod
  • 完整的 Docker 兼容性：命令语法与 Docker 类似
  • 集成 Buildah：支持构建容器镜像

Podman 广泛应用于容器化应用部署、开发测试环境、无根容器运行、安全容器环境等场景。

安装方法 ​

macOS 系统 ​

使用 Homebrew 安装 ​

bash
# 安装 Podman
brew install podman

# 初始化 Podman 机器（macOS 需要 VM）
podman machine init

# 启动 Podman 机器
podman machine start

# 验证安装
podman version

podman run hello-world

podman run -e VAR_NAME=value nginx

# 运行并设置容器名称
podman run --name my-container nginx

容器生命周期管理 ​

bash
# 列出运行中的容器
podman ps

# 列出所有容器（包括停止的）
podman ps -a

# 启动已停止的容器
podman start container_name_or_id

# 停止运行中的容器
podman stop container_name_or_id

# 重启容器
podman restart container_name_or_id

# 暂停容器
podman pause container_name_or_id

# 恢复暂停的容器
podman unpause container_name_or_id

# 删除容器
podman rm container_name_or_id

# 强制删除运行中的容器
podman rm -f container_name_or_id

# 删除所有停止的容器
podman container prune

# 进入运行中的容器
podman exec -it container_name_or_id bash

# 在容器中执行命令
podman exec container_name_or_id command

Rootless 模式操作 ​

配置 Rootless 模式 ​

bash
# 检查 Rootless 支持
podman info | grep rootless

# 运行 Rootless 容器
podman run --rm hello-world

# 在 Rootless 模式下运行后台服务
podman run -d --userns=keep-id -p 8080:80 nginx

# 设置 Rootless Podman 为默认
# 修改 ~/.bashrc 或 ~/.zshrc
export DOCKER_HOST=unix:///run/user/$(id -u)/podman/podman.sock

Podman Compose ​

使用 Podman Compose 替代 Docker Compose ​

bash
# 安装 Podman Compose
pip3 install podman-compose

# 或者从 GitHub 安装
curl -sSL https://raw.githubusercontent.com/containers/podman-compose/devel/podman_compose.py -o /usr/local/bin/podman-compose
chmod +x /usr/local/bin/podman-compose

# 使用 Podman Compose
podman-compose up

# 后台运行
podman-compose up -d

# 停止服务
podman-compose down

# 构建并启动
podman-compose up --build

常用操作示例 ​

1. 完整的 Web 应用部署 ​

bash
# 拉取镜像
podman pull nginx
podman pull mysql:8.0

# 运行数据库
podman run -d \
  --name mysql-db \
  -e MYSQL_ROOT_PASSWORD=rootpassword \
  -e MYSQL_DATABASE=webapp \
  -e MYSQL_USER=webuser \
  -e MYSQL_PASSWORD=webpass \
  -v mysql-data:/var/lib/mysql \
  mysql:8.0

# 运行 Web 应用
podman run -d \
  --name web-app \
  --link mysql-db:mysql \
  -p 8080:80 \
  nginx

安全配置 ​

安全最佳实践 ​

bash
# 运行非 root 容器
podman run --user 1000:1000 nginx

# 禁用特权模式
# 避免使用 --privileged 标志

# 只读根文件系统
podman run --read-only nginx

# 限制系统调用
podman run --security-opt=no-new-privileges nginx

# 使用用户命名空间
podman run --userns=private nginx

最佳实践 ​

  1. 安全性：优先使用 Rootless 模式，限制容器权限
  2. 资源管理：设置适当的资源限制，避免资源争用
  3. 存储管理：使用命名卷而非绑定挂载，便于管理
  4. 网络隔离：使用自定义网络而非默认网络
  5. 镜像管理：使用特定标签而非 latest，定期更新镜像
  6. 日志管理：配置适当的日志驱动和日志轮转
  7. 监控告警：监控容器状态和资源使用情况
