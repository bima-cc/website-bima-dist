关于我们 ​

北京必码科技工作室（简称"必码"），是一家专注于开源项目定制开发与技术服务的工作室。我们秉承"只写必要的代码"的核心理念，围绕企业级身份认证与微服务架构两大核心方向，提供轻量化、可落地的技术解决方案。

我们的理念 ​

必码坚持"小而精、专而深"的服务原则，聚焦开源生态，专注技术落地。我们深知企业在使用开源产品时面临的"适配难、定制难"痛点，凭借丰富的开源二次开发经验，从需求分析到交付运维，全程细致跟进，力求以最小的成本解决最实际的问题。

技术能力 ​

我们深耕 Java 后端技术生态，主要涵盖以下领域：

  • 身份认证与授权：基于 Apereo CAS 的企业级单点登录系统定制开发，覆盖 OAuth 2.0 + OIDC 双协议集成、自定义认证处理器与自定义返回值、JWT AccessToken 加密签名、多因素认证等场景
  • 微服务架构：Spring Boot 单体、Dubbo 分布式 RPC、Spring Cloud 微服务三种架构形态的设计、选型与落地实施，同一业务场景提供可直接对比的参考实现
  • 中间件集成：MyBatis、Elasticsearch、Kafka、MongoDB、RabbitMQ、Redis、RocketMQ、LDAP 等主流中间件的深度集成与调优，一站式完成接入调试、功能测试与可用性验证
  • AI 能力集成：Spring AI + Ollama 本地大模型、OpenAI API 双模式对接、SSE 流式实时聊天与文章创作、Chroma 文本向量嵌入、内置写作风格系统等多元化 AI 应用落地
  • 容器化与部署：Docker 多阶段构建、Jenkins CI/CD 流水线、Kubernetes 容器编排，内置标准化 DockerFile 与部署脚本，一键完成容器打包与自动化部署
  • Keycloak 扩展开发：基于 Keycloak SPI 的扩展开发，包括用户存储扩展、事件监听器扩展和国密算法扩展，支持多种数据库类型、消息队列系统和中国国家标准密码算法

业务服务 ​

CAS 单点登录系统 ​

基于 Apereo 基金会维护的 CAS（Central Authentication Service）项目，提供面向企业的轻量化定制开发服务，解决多系统统一身份认证问题。覆盖 5.3.x（JDK 8）、6.6.x（JDK 11）、7.3.x（JDK 21 / Spring Boot 3.x）三个大版本线，支持 Gradle 和 Maven 双构建体系，提供完整的开发基座、配置模板和部署方案。

核心能力： WAR 转 JAR 现代化架构升级 · JDBC + LDAP 双源认证 · 自定义认证处理器与自定义返回值 · OAuth 2.0 + OIDC 双协议认证 · JWT AccessToken 加密签名 · Redis 分布式票据注册表 · 安全加固（强制关闭默认账号） · 自定义启动类支持本地 Debug · 前端资源全量提取与品牌定制 · 国际化多语言 · 监控与日志体系 · Docker / Jenkins CI/CD 自动化部署

查看 CAS 版本文档 · 阅读 CAS 深度解析系列

智能中间件集成验证脚手架 ​

面向开发者打造的一站式微服务项目脚手架，深度融合 AI 能力与主流中间件集成方案。同一业务场景在 Spring Boot 单体、Dubbo RPC、Spring Cloud 微服务三种架构下的完整实现，为架构选型提供可直接对比的参考。基于 Spring Boot 3.5.12 + Java 17，内置 LayUI 可视化 Web 操作界面，零代码门槛即可完成各类中间件功能调试与 AI 能力在线演示。

核心能力： 8 大中间件集成验证（MyBatis / Elasticsearch / Kafka / MongoDB / RabbitMQ / Redis / RocketMQ / LDAP） · AI 全场景集成（Ollama + OpenAI 双模式、SSE 流式聊天与文章创作、Chroma 文本向量嵌入、内置写作风格系统） · OAuth 2.0 + OIDC 双协议统一认证 · 多数据源动态配置 · 全覆盖内置测试用例 · 多环境无缝适配 · Docker / Jenkins CI/CD · Maven 多模块工程化

查看脚手架文档 · 阅读 Smart-Scaffold 深度解析系列

Keycloak 扩展开发 ​

基于 Keycloak 开源项目，提供面向企业的扩展开发服务，解决 Keycloak 原生功能不足的问题。支持三种核心扩展类型：用户存储扩展、事件监听器扩展和国密算法扩展，满足企业在身份认证、事件处理和加密需求等方面的定制化需求。

核心能力： 多数据库类型支持（MySQL、SQL Server、Oracle、达梦、金仓、OceanBase、GaussDB） · 多消息队列集成（Kafka、RabbitMQ、RocketMQ） · 国密算法集成（SM2、SM3、SM4） · 灵活的扩展架构 · 完整的开发示例 · 标准化部署方案

查看 Keycloak 扩展开发文档
